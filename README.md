# PrintOffice — база знаний (веб-сервис)

Статический веб-сервис: структурированная база знаний проекта PrintOffice с кликабельным оглавлением, рендером разделов и поиском. Готов к публикации на GitHub Pages и открытию по техническому домену.

В базе — **только чистые бизнес-материалы**. Черновики, технические наброски и внутренние рабочие документы из публикации исключены.

## Что внутри

```
site/
├── manifest.mjs        что входит в базу и под какими названиями (оглавление)
├── content/            чистые исходники .md (самодостаточная копия)
├── assets/             стили и поиск (исходники)
├── build.mjs           генератор: content/ → docs/
├── sync.mjs            обновить content/ из рабочих .md (на машине разработчика)
├── serve.mjs           локальный просмотр
├── docs/               СОБРАННЫЙ САЙТ (его отдаёт GitHub Pages)
└── .github/workflows/  автосборка и деплой на Pages
```

## Локальный просмотр

```bash
npm install        # один раз
npm run build      # собрать docs/ из content/
npm run serve      # открыть http://localhost:8080
```

`npm run all` = `sync` + `build` (обновляет content/ из рабочих .md проекта и пересобирает; sync нужен только если работаете внутри общего проекта `context/`).

## Публикация на GitHub Pages

1. Создать репозиторий и запушить содержимое папки `site/` (она и есть корень репозитория).
2. В **Settings → Pages**:
   - вариант с автосборкой (рекомендуется): Source = **GitHub Actions** — деплой выполнит workflow `.github/workflows/deploy.yml` при каждом пуше в `main`;
   - либо без Actions: Source = **Deploy from a branch**, ветка `main`, папка **/docs**.
3. Технический домен: переименовать `CNAME.example` → `CNAME`, вписать домен (одной строкой), у регистратора добавить CNAME-запись на `<username>.github.io`. После этого база открывается по вашему домену.

## Как обновить содержимое

- Изменить нужный `.md` в `content/` (или в рабочем проекте и затем `npm run sync`).
- Состав и порядок разделов — в `manifest.mjs`.
- `npm run build` → закоммитить → Pages обновится автоматически.

## Что исключено из публикации

Черновики (`raw/`), технические документы (модель данных, API, стек, структура репозитория) и внутренние рабочие материалы (аудит, единая истина, журналы решений, дорожная карта, вопросы). При сборке также вырезаются служебные секции «Источник» и внутренние пометки — для клиента всё чисто.
