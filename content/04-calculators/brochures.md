# Калькулятор брошюр и многостраничной продукции

Спецификация модуля расчёта составной многостраничной продукции (брошюры на скобе, пружине, КБС, болтах): кратность страниц, раскладка, подбор пружины, пластиковые обложки и цветные подложки, расчёт себестоимости по блокам, валидация, маржа и чек-листы готовности.

---

## 1. Концепция: брошюра как композиция блоков

Брошюра — составное изделие, которое рассчитывается как сумма независимых блоков. Каждый блок имеет собственную бумагу/материал, цветность, раскладку и себестоимость.

| Блок | Обязательность | Содержание |
|------|----------------|------------|
| 📖 Обложка | опционально (можно отключить) | Печать (бумага, цветность), доп. услуги (ламинация, биговка). Тираж обложек = тираж брошюр |
| 📄 Внутренний блок | обязательно | Печать (бумага, цветность), количество страниц (кратно 2 или 4) |
| 🎨 Подложка (back cover) | опционально, только для пружины/болтов | Материал (картон, цвет), печать опциональна. Тираж = тираж брошюр |
| ✨ Материалы | опционально | Пластиковая обложка, защитная плёнка, картонная вкладка |
| 🔗 Сборка | обязательно | Тип переплёта (скоба/пружина/КБС), сторона крепления, визуализация |

### Финальная формула расчёта

```
Общая себестоимость =
    cover_cost          (если включён)
  + block_cost          (обязательно)
  + back_cover_cost     (если включён и тип = пружина)
  + materials_cost      (если включены)
  + binding_cost        (обязательно)

Цена для клиента = Общая себестоимость × (1 + маржа / 100)
```

Где:
- `cover_cost = sheets × (print_price + paper_price) + lamination_cost` (если есть)
- `block_cost = sheets × (print_price + paper_price)`
- `back_cover_cost = sheets × material_price` (без печати) или `sheets × (print_price + paper_price)` (с печатью)
- `materials_cost = material_price × quantity` (для каждого материала)
- `binding_cost = цена сборки` (фиксированная или зависит от тиража/толщины)

## 3. Раскладка и формулы расчёта блоков

### 3.1. Раскладка на печатном листе (общая для всех блоков)

Печатный лист по умолчанию — SRA3 (320×450 мм).

```javascript
function calculateLayout(format, orientation) {
  const formats = { A4:{w:210,h:297}, A5:{w:148,h:210}, A6:{w:105,h:148} };
  const sheet = { width:320, height:450 }; // SRA3
  const p = formats[format];
  const itemW = orientation === 'portrait' ? p.w : p.h;
  const itemH = orientation === 'portrait' ? p.h : p.w;
  // страниц на одной стороне листа
  const pagesPerSide = Math.floor(sheet.width/itemW) * Math.floor(sheet.height/itemH);
  const pagesPerSheet  = pagesPerSide * 2;   // двусторонняя печать
  const coversPerSheet = pagesPerSide;       // обложки/подложки — односторонне считается на сторону
  return { pagesPerSheet, coversPerSheet };
}
```

Базовые формулы:

```
# внутренний блок
sheets_per_brochure = ceil(total_pages / pages_per_sheet)
total_sheets        = sheets_per_brochure × quantity

# обложка / подложка
sheets = ceil(quantity / covers_per_sheet)

# себестоимость блока
cost = sheets × (print_price + paper_price)
```

### 3.2. Себестоимость каждого блока

```
# Обложка
cover_sheets   = ceil(quantity / covers_per_sheet)
cover_cost     = cover_sheets × (print_price + paper_price) + lamination_cost
lamination_cost = quantity × lamination_price_per_unit

# Внутренний блок
block_sheets   = ceil(pages / pages_per_sheet) × quantity
block_cost     = block_sheets × (print_price + paper_price)

# Подложка (картон, обычно без печати)
back_sheets    = ceil(quantity / covers_per_sheet)
back_cover_cost = back_sheets × material_price  (+ печать, если выбрана)

# Сборка
binding_cost   = price_per_unit(quantity) × quantity  # фиксированная или ступенчатая по тиражу
```

⚠️ ОТКРЫТЫЙ ВОПРОС: количество страниц на листе для A4 указано непоследовательно. В одном фрагменте `calculateLayout` для брошюры A5 даёт 8 страниц на листе (4 с каждой стороны), а упрощённая таблица в `BrochureCostCalculator.calculateLayout` задаёт фиксированно A4=4, A5=8, A6=16 стр/лист независимо от ориентации; для авторефератов на скобе зашито `pagesPerSheet = 2` для A4 «в сложенном виде». Нужно зафиксировать единый алгоритм раскладки.

## 4. Класс расчёта брошюры (полный, с учётом всех блоков)

Полный калькулятор собирает себестоимость из блоков-классов `InnerBlock`, `CoverBlock`, `BackCoverBlock`, `PlasticCoverBlock`, `BindingBlock`.

```javascript
class BrochureCalculator {
  async calculate(params) {
    const results = { blocks:{}, totalCost:0, errors:[] };

    // 1. Внутренний блок (обязательный)
    const block = await new InnerBlock().calculate({ format, orientation, pages, paperId:blockPaperId, colorId:blockColorId, quantity });
    results.blocks.inner = block; results.totalCost += block.totalCost;

    // 2. Обложка (опционально)
    if (coverEnabled) {
      const cover = await new CoverBlock().calculate({ format, orientation, paperId:coverPaperId, colorId:coverColorId, hasLamination:coverLamination, quantity });
      results.blocks.cover = cover; results.totalCost += cover.totalCost;
    }

    // 3. Подложка (только для пружины)
    if (bindingType === 'spring' && backCoverEnabled) {
      const back = await new BackCoverBlock().calculate({ format, orientation, materialId:backCoverMaterialId, color:backCoverColor, hasPrint:backCoverPrint, quantity });
      results.blocks.backCover = back; results.totalCost += back.totalCost;
    }

    // 4. Пластиковая обложка
    if (plasticCoverEnabled) {
      const plastic = await new PlasticCoverBlock().calculate(format, plasticCoverType, quantity);
      if (!plastic.success) results.errors.push(plastic.error);
      else { results.blocks.plasticCover = plastic; results.totalCost += plastic.finalCost; }
    }

    // 5. Сборка
    const binding = await new BindingBlock().calculate({ type:bindingType, pages, paperDensity, quantity });
    if (!binding.success) results.errors.push(binding.error);
    else { results.blocks.binding = binding; results.totalCost += binding.totalCost; }

    // 6. Маржа
    const margin = await this.getMargin(quantity);
    const finalPrice = results.totalCost * (1 + margin/100);

    return { success: results.errors.length===0, breakdown: results.blocks, totalCost: results.totalCost, margin, finalPrice, pricePerUnit: finalPrice/quantity };
  }

  // Маржа от тиража (значения из кода калькулятора)
  async getMargin(quantity) {
    if (quantity < 10)  return 100;   // < 10 шт
    if (quantity < 50)  return 80;    // 10–49 шт
    if (quantity < 100) return 60;    // 50–99 шт
    return 50;                        // 100+ шт
  }
}
```

⚠️ ОТКРЫТЫЙ ВОПРОС: ступени маржи в коде `getMargin` (`<10`→100%, `<50`→80%, `<100`→60%, иначе 50%) не совпадают со ступенями админ-таблицы маржи из финальной спецификации конструктора (см. §5.4: 1–10→100%, 11–50→80%, 51–100→70%, 101+→60%). В частности для тиража 50–100 шт код даёт 60%, а админ-таблица — 70%. Нужно выбрать единый источник наценки (жёстко зашитая функция vs настраиваемая таблица).

Внутренние «справочные» цены, использованные в примерах кода (себестоимость, не финальная цена):

- Печать (за лист): `1+0` = 5; `1+1` = 8; `4+0` = 20; `4+4` = 25 руб.
- Бумага блока (за лист): 80 г/м² = 2; 130 = 3; 170 = 4; 200 = 4.5 руб.
- Бумага обложки (за лист): 130 = 3; 200 = 4.5; 250 = 6; 300 = 8 руб.
- Ламинация (за шт): A4 = 15; A5 = 10; A6 = 7 руб.
- Картон подложки (за лист): чёрный = 12; белый = 10; красный/синий = 15 руб.
- Сборка (за шт): скоба = 15 (в авторефератах 5); пружина = по подбору; КБС = 50 руб.

### 5.4. Маржа и дополнительные наценки (админ-таблица конструктора)

Маржа применяется к общей себестоимости всех блоков и настраивается отдельной таблицей в конструкторе брошюр (вкладка «Маржа»). Эти ступени заданы в финальной спецификации конструктора и отличаются от зашитой в коде функции `getMargin` (см. §4).

| От (экз) | До (экз) | Тип наценки | Значение |
|----------|----------|-------------|----------|
| 1 | 10 | Процент | 100% |
| 11 | 50 | Процент | 80% |
| 51 | 100 | Процент | 70% |
| 101 | ∞ | Процент | 60% |

Дополнительные наценки (включаются опционально и суммируются к базовой марже):

| Опция | Наценка |
|-------|---------|
| Срочность (до 24 ч) | +50% |
| Сложная раскладка (разные макеты) | +30% |
| Пластиковая обложка | +20% |

Маржа едина для всей брошюры (применяется к сумме себестоимости блоков), а не к каждому блоку отдельно.

## 6. Валидация параметров

Валидация выполняется при вводе и блокирует расчёт при ошибках (`errors`), либо выдаёт предупреждения (`warnings`), предложения (`suggestions`) и альтернативы (`alternatives`).

### 6.1. Ограничения по формату

| Параметр | Значение |
|----------|----------|
| Минимальный размер для ручной резки | 20×20 мм |
| Меньше 30×30 мм | только вырубка на плоттере |
| Минимальный формат брошюры | 50×50 мм (в части правил 50×70 мм) |
| Максимум для цифровой печати | 320×450 мм |
| Разворот не помещается на SRA3 | предупреждение «печать на большем листе / широкоформат» |

Логика разворота:
```
spread_width  = (orientation==='vertical') ? format.width*2  : format.height*2
spread_height = (orientation==='vertical') ? format.height   : format.width
fits = spread_width <= printableWidth && spread_height <= printableHeight
```
Если не помещается — предлагаются альтернативы: повернуть разворот, использовать больший формат бумаги, разбить на несколько листов (дороже).

### 6.2. Ограничения по страницам

```
multiple = (bindingType==='staple') ? 4 : 2
pages % multiple === 0           # иначе ошибка кратности
minPages <= pages <= maxPages    # иначе ошибка + предложение альтернативы
```

| Тип | minPages | maxPages | maxSheets |
|-----|----------|----------|-----------|
| Скоба | 8 | 64 | 32 |
| Пружина | 4 | 200 | 100 |
| КБС | 20 | 400 | — |

Толщина корешка для скобы > 5 мм → предупреждение «рассмотрите пружину/КБС».

### 6.3. Ограничения по бумаге (биговка/фальцовка)

Плотная бумага плохо фальцуется и краска осыпается — нужна биговка (а для очень плотной — биговка перед фальцовкой).

| Плотность, г/м² | Фальцовка | Биговка | Примечание |
|-----------------|-----------|---------|------------|
| ≤ 80 | да | нет | |
| 130 | да | нет | |
| 170 | да | рекомендуется | порог `scoringRequired` |
| 200 | нет | да | только биговка |
| 250 | нет | да | макс. плотность для скобы |
| 300 | нет | да | биговка + осторожно |
| 350 | нет | да | биговка обязательна |

Пороги по типам переплёта:
- Скоба: макс. плотность 250 г/м²; биговка с 170 г/м². Плотность обложки > 300 г/м² → ошибка (слишком толстая для скобы).
- Пружина: макс. плотность 300 г/м²; биговка с 200 г/м².
- КБС: плотность блока 80–300 г/м².

Прочие проверки бумаги:
- Обложка < 150 г/м² → предупреждение «недостаточно жёсткая, рекомендуется от 200 г/м²».
- Скоба: очень плотная обложка (> 250) + тонкий блок (< 100) → предупреждение «плохо скрепляется скобой».

### 6.4. Технологические ограничения

- Мелкий формат (< 30×30 мм) при тираже > 10 шт → рекомендуется вырубка на плоттере (множитель цены ×1.5, времени ×0.3) или штанц-форма.
- Офсетная печать при тираже < 100 шт → предупреждение «рекомендуется цифровая».
- Формат > печатного листа при цифре → ошибка + переход на широкоформатную печать.

### 6.5. Автоподбор альтернатив (цепочки замен)

Цепочка по умолчанию: **скоба → пружина → КБС**.

```javascript
findAlternativeForManyPages(pages, type) {
  if (type==='staple' && pages>64)  return { type:'spring', url:'/catalog/brochures?type=spring' };
  if (type==='spring' && pages>200) return { type:'kbs',    url:'/catalog/brochures?type=kbs' };
  return null;
}
```

При выборе альтернативы выполняется перерасчёт цены и сроков; страницы округляются до кратности нового типа.

## 7. Автоматический расчёт максимального количества страниц

Максимум страниц зависит от плотности бумаги, типа переплёта и допустимой толщины корешка.

```javascript
calculateMaxPages(paperDensity, bindingType) {
  const sheetThickness = paperDensity * 0.0011;            // толщина листа, мм
  const maxSpineThickness = { staple:5, spring:15, kbs:25 }; // макс. корешок, мм
  const maxThickness = maxSpineThickness[bindingType] || 10;
  const maxSheets = Math.floor(maxThickness / sheetThickness);
  let maxPages = maxSheets * 2;                            // лист = 2 страницы
  const multiple = bindingType==='staple' ? 4 : 2;
  maxPages = Math.floor(maxPages / multiple) * multiple;   // кратность
  const minPages = bindingType==='staple' ? 8 : 4;
  return { minPages, maxPages, maxSheets, spineThickness:(maxSheets*sheetThickness).toFixed(1) };
}
```

Пример: 130 г/м² → толщина листа 0.143 мм; пружина (корешок 15 мм) → 104 листа → 208 страниц (кратно 2 = 208).

В админке поддерживается переключатель «использовать автоматический расчёт / ручной лимит min-max страниц».

⚠️ ОТКРЫТЫЙ ВОПРОС: максимум страниц для скобы задан жёстко как 64 в правилах валидации, но автоматический расчёт `calculateMaxPages` при допустимой толщине корешка 5 мм и тонкой бумаге выдаёт значительно больше. Нужно решить, что приоритетнее — жёсткий лимит из настроек или автоматический расчёт по толщине.

## 8. Подбор пружины

### 8.1. Принцип

Производитель указывает ёмкость пружины для офисной бумаги 80 г/м². Для другой плотности выполняется пересчёт по толщине листа. В параметрах пружины учитываются материал (металл/пластик) и цвет.

```
sheet_thickness  = paperDensity × 0.0011          # мм на лист
spine_thickness  = (pages / 2) × sheet_thickness   # толщина корешка, мм
required_thickness = spine_thickness × safety_margin + spine_margin
```

Коэффициент запаса материала `safety_margin` (мультипликативный): металл = 1.0, пластик = 1.2 (нужен запас +10–20%; `max_sheets_factor`: металл 1.0, пластик 0.9).

Аддитивный запас на корешок `spine_margin` задаётся в админ-формуле и по умолчанию = **2 мм**: при подборе пружины он прибавляется к рассчитанной толщине корешка (`required_thickness = spine_thickness + spine_margin`, формула из `ConfigurableRuleEngine.selectSpring`). Запас редактируется в админке вместе с формулами толщины бумаги (`density × 0.0011`) и макс. толщины для скобы (5 мм).

### 8.2. Пересчёт ёмкости под плотность

```
base_density = 80
ratio        = thickness(target_density) / thickness(80)
adjustedMin  = round(base_pages_min / ratio)
adjustedMax  = round(base_pages_max / ratio)
fits         = target_pages in [adjustedMin, adjustedMax]
```

### 8.3. Толщина бумаги по плотности (предустановленная таблица)

Предустановленная таблица толщины по плотности (80=0.10, 130=0.16 … 400=0.48), формула `density × 0.0011` (используемая в §7 и §8.1) и сведённое расхождение между ними (для 80 г/м²: 0.10 мм по таблице vs 0.088 мм по формуле; для 130 г/м²: 0.16 мм vs 0.143 мм) — в каноническом справочнике толщин: [Оборудование и нормативы, раздел 8.1 «Толщина и вес бумаги»](../06-production/equipment.md#81-толщина-и-вес-бумаги-канонический-справочник-толщин).

### 8.4. Справочник пружин (пример)

| Диаметр | Страницы (80 г/м²) | Толщина корешка | На складе | Цена |
|---------|--------------------|-----------------|-----------|------|
| 6 мм | 10–20 | 2.0–3.5 мм | 500 шт | 15 руб |
| 8 мм | 21–35 | 3.6–5.5 мм | 320 шт | 18 руб |
| 10 мм | 36–50 | 5.6–7.5 мм | 150 шт | 22 руб |
| 12 мм | 51–70 | 7.6–9.5 мм | 80 шт | 28 руб |

Параметры пружины в БД: диаметр, материал (`metal`/`plastic`), цвет, базовые min/max листов и толщина для 80 г/м², коэффициент пересчёта (по умолчанию 0.0011), остаток на складе, точка заказа, цена, поставщик.

Цвета: металл — чёрный, белый, серебро, золото, синий; пластик — чёрный, белый, прозрачный, красный, синий, зелёный.

Алгоритм подбора: рассчитать толщину корешка → применить запас материала → найти пружины, в диапазон которых попадает требуемая толщина/количество → отсортировать по приоритету (близость к оптимуму, предпочтение металлу +10, штраф при остатке < 10 шт) → отфильтровать по материалу/цвету → проверить наличие на складе. Если пружина не найдена — предложить КБС.

### 8.5. Производственное задание (ТЗ) по сборке

В ТЗ передаются: диаметр, материал и цвет пружины, тираж, толщина корешка, пересчитанные лимиты под текущую плотность, предупреждение о нехватке на складе.

## 9. Пластиковая обложка

Пластиковые обложки используются в готовых форматах A4/A3 (реже A5) — в ~90% случаев резка не требуется. Резка нужна только для нестандартного размера.

### 9.1. Готовые форматы и цены

| Формат | Размер, мм | Тип | Цена, руб | На складе | Подходит для |
|--------|------------|-----|-----------|-----------|--------------|
| A4 | 210×297 | прозрачный | 45 | 500 | A4, A5 |
| A4 | 210×297 | матовый | 55 | 300 | A4, A5 |
| A4 | 210×297 | цветной (красный) | 65 | 100 | A4 |
| A4 | 210×297 | цветной (синий) | 65 | 100 | A4 |
| A5 | 148×210 | прозрачный | 30 | 500 | A5, A6 |
| A5 | 148×210 | матовый | 40 | 300 | A5, A6 |
| A5 | 148×210 | цветной (красный) | 45 | 100 | A5 |
| A3 | 297×420 | прозрачный | 85 | 200 | A3, A4 |
| A3 | 297×420 | матовый | 95 | 150 | A3, A4 |

Типы пластика: прозрачный (базовая цена), матовый (+10 руб), цветной (по запросу/индивидуально).

### 9.2. Скидки по тиражу

| От (шт) | До (шт) | Скидка |
|---------|---------|--------|
| 1 | 49 | 0% |
| 50 | 99 | 5% |
| 100 | ∞ | 10% |

### 9.3. Расчёт и склад

```
total_cost = price × quantity
discount   = total_cost × discount_percent / 100
final_cost = total_cost − discount
available_stock = stock_quantity − reserved_quantity   # проверка наличия
```

Складской учёт: проверка наличия при расчёте, резервирование при оформлении заказа (`reserve`), списание при отгрузке (`writeOff`), освобождение резерва при отмене (`releaseReservation`), пополнение (`restock`), история движений (`plastic_covers_stock_history`).

### 9.4. Резка нестандарта

Если нужного готового формата нет — расчёт резки:
```
covers_per_sheet = floor(sheet_w / target_w) × floor(sheet_h / target_h)   # лист обычно 320×450
sheets        = ceil(quantity / covers_per_sheet)
material_cost = sheets × material.cost_per_unit
cutting_cost  = quantity × perimeter_m × cutting_time_per_meter × cost_per_minute
total_cost    = (material_cost + cutting_cost) × custom_cut_markup           # наценка за нестандарт, по умолчанию ×1.2 (+20%)
```
Параметры по умолчанию: минимальный размер резки 50×50 мм; стоимость резки 30 руб/мин; время резки 0.5 мин/метр; резать специальным ножом, скорость снижена на 30%.

## 10. Подложка (back cover)

Задняя обложка для брошюр на пружине: обеспечивает жёсткость, может быть с печатью или без.

### 10.1. Материалы

| Материал | Плотность/толщина | Цвета | Цена (себест./продажа), руб/лист |
|----------|-------------------|-------|----------------------------------|
| Картон | 250 г/м² | чёрный | 8 / 12 |
| Картон | 250 г/м² | белый | 7 / 10 |
| Картон | 250 г/м² | красный | 10 / 15 |
| Картон | 300 г/м² | чёрный | 10 / 15 |
| Картон | 300 г/м² | белый | 9 / 13 |
| Пластик | 0.5 мм | прозрачный | 18 / 25 |

Цены печати на подложке (за лист): A4 `1+0`=10, `4+0`=25; A5 `1+0`=7, `4+0`=18; A6 `1+0`=5, `4+0`=12.

Коды цветов для отображения: чёрный `#1a1a1a`, белый `#f5f5f5`, красный `#e53935`, синий `#1e88e5`, зелёный `#43a047`, прозрачный `#e0e0e0`.

### 10.2. Расчёт

```
sheets         = ceil(quantity / covers_per_sheet)   # раскладка как у обложки (SRA3)
material_cost  = selling_price × sheets
print_cost     = print_price × sheets                # если hasPrint
total_cost     = material_cost + print_cost
```
Проверки: цвет должен входить в список доступных для материала; наличие на складе (`stock − reserved >= sheets`). Поддерживаются резервирование и списание, как у пластиковых обложек.

Ступенчатая цена сборки на пружине:

| 1 | 10 | 30 руб/шт |

Примеры: тираж 5 шт → 150 руб; 20 шт → 500 руб; 100 шт → 2000 руб.

### 11.2. Переопределение для конкретного конструктора

## 12. Конструктор авторефератов/каталогов на скобе (комбинированная печать)

Экономичный вариант брошюры на скобе: дешёвый ч/б основной блок + цветная обложка + опциональные цветные вставки. Поддерживает разную бумагу и цветность для разных частей.

| Блок | Бумага | Печать | Кратность |
|------|--------|--------|-----------|
| Обложка | цветная (напр. 200 г/м²) | 4+0 или 4+4, опц. ламинация | — |
| Основной блок | офсетная 80 г/м² (дёшево) | 1+0 или 1+1 (ч/б) | 4 |
| Цветные вставки (опц.) | мелованная 130 г/м² | 4+4 | 4, в пределах основного блока |

Расположение цветных вставок: в начале (`start`), в конце (`end`), внутри в разбивку (`mixed`).

Расчёт каждого блока: `sheets = ceil(pages / pages_per_sheet)` (для блоков) или `ceil(quantity / covers_per_sheet)` (обложка), затем `cost = sheets × (paper_price + print_price)`. Сборка на скобу — фиксированно 5 руб/шт.

### Пример

Вход: печать A4; обложка цветная 200 г/м² 4+0 без ламинации; основной блок офсет 80 г/м² 1+1, 32 стр; цветные вставки мелованная 130 г/м², 4 стр в начале; тираж 50 шт.

| Блок | Себестоимость |
|------|---------------|
| Обложка | 200 руб |
| Основной блок | 1280 руб |
| Цветные вставки | 150 руб |
| Сборка | 250 руб |
| **ИТОГО** | **1880 руб (37.60/шт)** |

## 14. Фронтенд: универсальная страница и UX

- **Одна страница `/catalog/brochures`** с переключением типов переплёта через иконки (скоба/пружина/КБС/болты). При переключении сохраняется контекст: предыдущие параметры подгружаются из истории (localStorage, последние 10 переходов), URL обновляется для SEO (`?type=staple&format=a5&pages=20&orientation=vertical`).
- **Возможность отдельных страниц** под каждый тип переплёта для SEO + перелинковка («похожий товар»: каталог на скобе → каталог на пружине) с сохранением истории расчёта при переходе.
- **Визуализация (SVG)**: схема брошюры в сложенном и развёрнутом виде с размерами по высоте/ширине; автоматически меняется при выборе формата/ориентации; показывает сторону крепления скоб/пружины (книжная → слева, альбомная → сверху).
- **Проверка влезания разворота** на печатный лист при вводе своего формата с предложением альтернатив (повернуть / больший лист / широкоформат).
- **Отображение валидации**: ошибки блокируют расчёт, предупреждения с кнопкой авто-исправления (например, добавить биговку), карточки альтернатив с переходом и оценкой цены.
- **SEO**: Schema.org `Product`/`AggregateOffer` (lowPrice/highPrice по типу и формату), микроразметка для всех типов.

## 16. Интеграция с производством (ТЗ)

В производственное задание передаются:
- **Печать:** для блока и обложки — количество листов, имозиция (`N стр/лист`), бумага, цветность.
- **Послепечать:** фальцовка (`N страниц`), сборка (на скобу/пружину/КБС), обрезка с трёх сторон, ламинация обложки (если есть).
- **Сборка:** для пружины — диаметр, материал, цвет пружины, толщина корешка, пересчитанные под плотность лимиты, предупреждение о нехватке на складе.
- **Спецматериалы:** для пластиковой обложки — инструкция «специальный нож, скорость реза снижена»; порядок сборки «обложка (пластик) → блок → подложка (картон)»; предупреждение «пластик царапается».
- **Контроль качества:** совпадение цветов обложки и блока, ровность корешка, отсутствие дефектов печати.

⚠️ ОТКРЫТЫЙ ВОПРОС: складской учёт остатков (пружины, пластиковые обложки, картон подложки) на момент проектирования реализован заглушками (`InventoryManager`, `FutureInventorySystem`). Полноценные резервирование/списание/автозаказ откладываются до этапа построения склада.

## Открытые вопросы (сводно)

1. Непоследовательное число страниц на печатном листе для A4 (8 vs фиксированная таблица vs `2` для авторефератов «в сложенном виде») — нужен единый алгоритм раскладки.
2. Максимум страниц для скобы: жёсткий лимит 64 vs автоматический расчёт по толщине корешка — что приоритетнее.
3. Два несовместимых способа расчёта толщины листа: `density × 0.0011` vs предустановленная таблица `paper_density_thickness` (80 г/м²: 0.088 vs 0.10 мм) — выбрать единый источник.
4. Ступени маржи в коде `getMargin` (`<10`→100%, `<50`→80%, `<100`→60%, иначе 50%) не совпадают с админ-таблицей маржи конструктора (1–10→100%, 11–50→80%, 51–100→70%, 101+→60%) — для тиража 50–100 шт расхождение 60% vs 70%. Нужен единый источник наценки.
5. Складской учёт остатков реализован заглушками; полноценная интеграция отложена до построения склада.
