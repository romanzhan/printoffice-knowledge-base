Оборудование и нормативы

Справочник оборудования типографии PrintOffice (печатное и послепечатное), его характеристик и нормативов времени/скоростей, а также универсальный конструктор оборудования и связанные справочники материалов, форматов и цветности. Документ описывает, как из набора параметров и формул собирается расчёт времени производственных операций без участия программиста.

1. Базовые справочники (статус)#

Базовые справочники — это «сырые данные» без привязки к конкретным услугам. Они переиспользуются в конструкторах услуг.

Справочник Описание Статус
Оборудование Принтеры, резаки, плоттеры, ламинаторы с их характеристиками Настроено
Материалы Бумага, плёнки, пружины, картон, самоклейка (каждый вариант — отдельная запись) Настроено
Форматы A4, A5, A6, A3, SRA3, евро, нестандартные Настроено
Цветность 1+0, 1+1, 4+0, 4+4 Настроено
Операции Ламинация, биговка, фальцовка, резка Настроено
Типы услуг Цифровая печать, офсет, широкоформат Настроено

1.1. Структура базовых справочников (SQL)#

-- 1.1. Принтеры (оборудование)
CREATE TABLE printers (
    id INT PRIMARY KEY,
    name VARCHAR(255),
    slug VARCHAR(100),
    type VARCHAR(50),              -- 'color' или 'bw'
    max_format_width INT,
    max_format_height INT,
    speed_ppm INT,                 -- скорость, листов в минуту
    cost_per_hour DECIMAL(10,2),   -- стоимость часа работы
    is_active BOOLEAN
);

-- 1.2. Материалы (бумага, плёнка и т.д.)
CREATE TABLE materials (
    id INT PRIMARY KEY,
    name VARCHAR(255),
    category VARCHAR(50),          -- 'paper', 'film', 'binding' и т.д.
    unit VARCHAR(20),              -- 'лист', 'пачка', 'кв_м', 'рулон'
    cost_per_unit DECIMAL(10,2),   -- закупочная цена
    stock_balance INT              -- остаток на складе
);

-- 1.3. Форматы (стандартные и нестандартные)
CREATE TABLE formats (
    id INT PRIMARY KEY,
    name VARCHAR(50),              -- 'A4', 'A5', 'SRA3', 'custom'
    width INT,
    height INT,
    is_standard BOOLEAN DEFAULT true,
    description VARCHAR(255)
);

-- 1.4. Цветность (типы печати)
CREATE TABLE color_types (
    id INT PRIMARY KEY,
    code VARCHAR(10),              -- '1+0', '1+1', '4+0', '4+4'
    description VARCHAR(255)
);

-- 1.5. Технологические операции
CREATE TABLE tech_operations (
    id INT PRIMARY KEY,
    name VARCHAR(100),             -- 'резка', 'биговка', 'фальцовка', 'ламинация'
    unit VARCHAR(20),              -- 'операция', 'минута', 'лист'
    cost_per_unit DECIMAL(10,2),   -- базовая стоимость
    time_per_unit INT              -- время в минутах на единицу
);

-- 2.1. Цены печати (принтер + материал + формат + цветность)
CREATE TABLE print_prices (

CREATE TABLE printer_operations (

    time_multiplier DECIMAL(3,2) DEFAULT 1.0   -- коэффициент к времени

2. Печатное оборудование: Konica C6085 и C3070L#

Этот раздел — единственный владелец матрицы скоростей печати и коэффициентов форматов. Калькуляторные файлы (../04-calculators/cutting-lamination-services.md, ../04-calculators/imposition-bleeds.md) ссылаются сюда за значениями скоростей и не дублируют таблицы.

2.1. Матрица скоростей печати (printer_speed_matrix)#

Скорость печати зависит от модели принтера, плотности и типа бумаги (обычная/глянец) и формата. За базу принят A4; хранится в таблице printer_speed_matrix.

Принтер Плотность, г/м² Тип Формат Скорость, стр/мин
C6085 52–400 любой A4 85
C6085 52–400 любой A3 85
C6085 52–400 любой SRA3 85
C3070L 62–216 обычная A4 71
C3070L 62–216 глянец A4 50
C3070L 217–350 обычная A4 50
C3070L 217–350 глянец A4 36
C3070L 52–61 любой A4 недоступно
C3070L 350–400 любой A4 недоступно

Ключевые выводы из матрицы:

  • Konica C6085 держит постоянную скорость 85 стр/мин во всём диапазоне плотностей 52–400 г/м² и для всех форматов (A4, A3, SRA3) — большой формат не снижает скорость, просто содержит больше изделий на лист.
  • Konica C3070L имеет переменную скорость, зависящую от плотности и типа поверхности: 71 → 50 → 36 стр/мин; недоступна для очень тонкой (52–61) и очень плотной (350–400) бумаги.

⚠️ ОТКРЫТЫЙ ВОПРОС: для C3070L матрица скоростей приведена только для формата A4. Скорости для A3/SRA3 на C3070L не заданы.

CREATE TABLE printer_speed_matrix (
    id INT PRIMARY KEY AUTO_INCREMENT,
    printer_id INT NOT NULL,
    paper_density_from INT,           -- минимальная плотность (г/м²)
    paper_density_to INT,             -- максимальная плотность (г/м²)
    paper_type VARCHAR(50),           -- 'обычная','глянец','любая'
    format_id INT,                    -- ссылка на formats (NULL = все)
    format_category VARCHAR(20),      -- 'A4','A3','SRA3','все'
    speed_ppm INT NOT NULL,           -- листов в минуту
    notes VARCHAR(255),
    valid_from DATE DEFAULT '2026-01-01',
    valid_to DATE NULL,
    FOREIGN KEY (printer_id) REFERENCES printers(id),
    FOREIGN KEY (format_id) REFERENCES formats(id)
);

2.2. Коэффициенты форматов (format_speed_coefficients)#

Производительность указывается для A4, поэтому для других форматов хранится коэффициент пересчёта относительно A4 (1.0 = как A4; >1.0 = медленнее; <1.0 = быстрее):

Принтер Формат Коэффициент Примечание
C6085 A4 1.0 базовый формат
C6085 A5 1.0 A5 на том же листе A4
C6085 A3 1.0 A3 = два A4, но скорость та же
C6085 SRA3 1.0 SRA3 чуть больше A3
C3070L A4 1.0 базовый формат
C3070L A3 1.2 A3 медленнее из-за большего размера

Итоговая скорость для формата: speed = baseSpeed_A4 / coefficient (при коэффициенте >1 скорость снижается).

Расчёт времени печати (класс PrintTimeCalculator, формула время = листы × копии / скорость + приладка) описан в калькуляторе послепечатных операций — см. ../04-calculators/cutting-lamination-services.md, раздел 5.

2.3. Спецформаты для C6085#

Помимо стандартных форматов, для Konica C6085 поддерживаются спецформаты увеличенной длины:

  • 330×487 мм
  • 330×762 мм

3. Универсальный конструктор оборудования#

Задача конструктора — гибко описывать любое оборудование с его характеристиками и зависимостями (влияющими на скорость работ и расчёт времени) без привлечения программиста для каждого нового станка или зависимости.

Каждый конструктор привязан к странице сайта: один конструктор = своя страница, страниц может быть несколько.

3.1. Четыре уровня конструктора#

УРОВЕНЬ 1: Категории оборудования
  ├── Печатное оборудование (принтеры)
  ├── Послепечатное оборудование
  │    ├── Резаки (гильотинные)
  │    ├── Ламинаторы (пакетные/рулонные)
  │    ├── Плоттеры
  │    ├── Биговальные/фальцовальные машины
  │    └── ...
  └── Вспомогательное оборудование

УРОВЕНЬ 2: Типовые параметры для категории
  ├── Для каждой категории свой набор параметров
  ├── Типы параметров: числовые, списковые, логические
  └── Единицы измерения (мм, мин, % и т.д.)

УРОВЕНЬ 3: Конкретные модели оборудования
  ├── Заполняются значениями параметров
  └── Привязка к материалам и зависимостям

УРОВЕНЬ 4: Правила расчёта времени
  ├── Формулы, использующие параметры
  └── Зависимости от материалов/форматов

3.2. Параметры по категориям (требования)#

  • Резак (гильотинный): скорость реза, макс. ширина реза, глубина реза, высота стопы.
  • Ламинатор пакетный: максимальный формат; температурные характеристики (влияют на толщину плёнки и вид плёнки — матовая/глянец); скорость ламинации, зависящая от плотности и формата плёнки.
  • Ламинатор рулонный: максимальная ширина ламинации, минимальная ширина, температура, плотность плёнки (влияет на скорость ламинации).
  • Плоттер: максимальный формат, скорость реза, давление ножа на материалах разной плотности.

У одного оборудования есть привязка к материалу, у другого её может не быть. Скорость выполнения работ может зависеть от формата и/или плотности материала.

3.3. Структура БД конструктора#

-- 1. Категории оборудования
CREATE TABLE equipment_categories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,        -- "Резаки гильотинные"
    code VARCHAR(50) UNIQUE,           -- "guillotine_cutters"
    description TEXT,
    icon VARCHAR(255),                 -- иконка для интерфейса
    sort_order INT DEFAULT 0,
    is_active BOOLEAN DEFAULT true
);

-- 2. Типы параметров для категорий
CREATE TABLE equipment_param_templates (
    id INT PRIMARY KEY AUTO_INCREMENT,
    category_id INT NOT NULL,
    param_name VARCHAR(100) NOT NULL,  -- "Макс. ширина реза"
    param_code VARCHAR(50) NOT NULL,   -- "max_cut_width"
    param_type ENUM('number','string','boolean','select') DEFAULT 'number',
    unit VARCHAR(20),                  -- "мм", "мин", "г/м²"
    select_options JSON,               -- ["глянец","мат","софт-тач"] (для типа 'select')
    affects_time BOOLEAN DEFAULT false,-- критичен ли параметр для расчёта времени
    is_required BOOLEAN DEFAULT true,
    placeholder VARCHAR(255),
    sort_order INT DEFAULT 0,
    FOREIGN KEY (category_id) REFERENCES equipment_categories(id),
    INDEX idx_category (category_id)
);

-- 3. Конкретные модели оборудования
CREATE TABLE equipment_models (
    id INT PRIMARY KEY AUTO_INCREMENT,
    category_id INT NOT NULL,
    name VARCHAR(255) NOT NULL,        -- "Boway BW-4908 V9"
    manufacturer VARCHAR(100),         -- "Boway"
    model_year INT,
    parameters JSON NOT NULL,          -- {"max_cut_width":490,"max_stack_height":80}
    compatible_materials JSON,         -- ["paper_density:80-300","film_type:any"]
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES equipment_categories(id),
    INDEX idx_category_model (category_id, is_active)
);

-- 4. Формулы расчёта времени
CREATE TABLE equipment_time_formulas (
    id INT PRIMARY KEY AUTO_INCREMENT,
    equipment_id INT NOT NULL,
    applies_to VARCHAR(50),            -- "all","material_specific","format_specific"
    material_condition JSON,           -- {"density_from":80,"density_to":300}
    format_condition JSON,             -- {"width_from":0,"height_from":0}
    formula_text TEXT NOT NULL,        -- "base_time + (cuts * time_per_cut * (stack_height/80))"
    available_variables JSON,          -- ["cuts","stack_height","sheets"]
    result_unit VARCHAR(20) DEFAULT 'minutes',  -- "minutes","seconds"
    priority INT DEFAULT 100,          -- какую формулу применять первой
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (equipment_id) REFERENCES equipment_models(id)
);

-- 5. Зависимости скорость/материал (матрица скоростей)
CREATE TABLE equipment_speed_matrix (
    id INT PRIMARY KEY AUTO_INCREMENT,
    equipment_id INT NOT NULL,
    material_type VARCHAR(50),         -- "paper","film","any"
    material_density_from INT,
    material_density_to INT,
    material_finish VARCHAR(50),       -- "glossy","matte","any"
    format_width INT,                  -- 0 = любой
    speed_value DECIMAL(10,2) NOT NULL,
    speed_unit VARCHAR(20) DEFAULT 'mm/sec',  -- "mm/sec","sheets/min","m/min"
    notes TEXT,
    FOREIGN KEY (equipment_id) REFERENCES equipment_models(id),
    INDEX idx_equipment_material (equipment_id, material_type)
);

3.4. Привязка конструктора к оборудованию и операциям#

-- Связь конструкторов с оборудованием
CREATE TABLE constructor_equipment (
    id INT PRIMARY KEY AUTO_INCREMENT,
    constructor_id INT NOT NULL,
    equipment_id INT NOT NULL,
    usage_type ENUM('primary','optional','alternative') DEFAULT 'primary',
    priority INT DEFAULT 100,
    default_settings JSON,             -- {"use_for":"all","auto_select":true}
    FOREIGN KEY (constructor_id) REFERENCES service_constructors(id),
    FOREIGN KEY (equipment_id) REFERENCES equipment_models(id),
    UNIQUE KEY unique_constructor_equipment (constructor_id, equipment_id)
);

-- Связь конструкторов с операциями
CREATE TABLE constructor_operations (
    id INT PRIMARY KEY AUTO_INCREMENT,
    constructor_id INT NOT NULL,
    operation_id INT NOT NULL,         -- ссылка на tech_operations
    equipment_id INT,                  -- конкретное оборудование или NULL (любое)
    default_params JSON,               -- {"bleed":3,"gap":0}
    is_default BOOLEAN DEFAULT false,
    is_optional BOOLEAN DEFAULT true,
    sort_order INT DEFAULT 0,
    FOREIGN KEY (constructor_id) REFERENCES service_constructors(id),
    FOREIGN KEY (operation_id) REFERENCES tech_operations(id),
    FOREIGN KEY (equipment_id) REFERENCES equipment_models(id)
);

4.1. Резак гильотинный Boway BW-4908 V9#

Шаблон параметров категории:

Параметр Код Тип Единица Влияет на время Обязателен
Макс. ширина реза max_cut_width number мм да да
Макс. глубина реза max_cut_depth number мм да да
Макс. высота стопы max_stack_height number мм да да
Скорость реза cut_speed number мин/рез да да

Характеристики модели Boway BW-4908 V9 (Boway, 2022):

Параметр Значение
Макс. ширина реза 490 мм
Макс. глубина реза 450 мм
Макс. высота стопы 80 мм
Скорость реза 0.5 мин/рез
Точность 0.1 мм
Совместимые материалы бумага плотностью 60–400 г/м²

Формула расчёта времени резки (applies_to = all, результат в минутах):

time = setup_time + (stacks * cuts * cut_speed * (1 + (stack_height / max_stack_height) * 0.5))

Доступные переменные: setup_time, stacks, cuts, cut_speed, stack_height, max_stack_height.

Полный вывод формулы, коэффициент уплотнения 0.85, расчёт высоты и числа стоп, число резов cutsForGrid(cols, rows) и примеры — в каноническом описании: Резка, ламинация и доп. услуги, раздел 2 «Гильотинная резка». Таблица толщины бумаги по плотности (80=0.10 … 400=0.48) и формула density × 0.0011 — в каноническом справочнике толщин ниже, раздел 8.1. Здесь приведена только карточка оборудования (характеристики модели).

Матрица скорости реза по плотности бумаги (пример из интерфейса настройки):

Материал Плотность, г/м² Скорость, мин/рез
Бумага 80–150 0.5
Бумага 150–250 0.4
Бумага 250–400 0.3

Параметр Значение

Макс. температура 130 °C
Типы плёнки глянец, мат
Совместимые материалы плёнка пакетная A3, A4

Матрица скорости ламинации (зависит от плотности и поверхности плёнки):

Тип Плотность плёнки, мкм Поверхность Ширина, мм Скорость, м/мин Примечание
film 75–100 глянец 210 0.9 Тонкая глянцевая плёнка
film 100–125 глянец 210 0.7 Средняя глянцевая
film 125–175 глянец 210 0.5 Толстая глянцевая
film 75–125 мат 210 0.6 Матовая плёнка

Примечание о столбце «Ширина, мм»: значение 210 — это поле format_width из таблицы скоростей (equipment_speed_matrix), которое в конструкторе задаёт условие «формат не уже указанной ширины» для применения строки скорости (210 ≈ ширина A4). Это не физический размер пакета: реальный размер пакета A3 по разделу 7.3 — 303×426 мм, A4 — 223×315 мм. То есть format_width = 210 в матрице — параметр сопоставления формата, а не габарит плёнки.

Дополнительные временные параметры (штрафы на операции). Помимо скорости перемещения (макс. 800 мм/сек) и скорости реза, время плоттерной резки зависит от штрафов на повороты и переходы ножа. Эти значения — характеристики оборудования и используются в формуле времени:

8. Расчётные параметры материалов (нормативы и формулы)#

8.1. Толщина и вес бумаги (канонический справочник толщин)#

Это единый справочник толщины бумаги по плотности для всей системы. На него ссылаются все расчёты, использующие толщину листа: высота стопы при гильотинной резке (Резка, ламинация и доп. услуги, раздел 2) и подбор пружины / автоёмкость брошюры (Калькулятор брошюр, раздел 8).

Толщина листа задаётся двумя способами.

(1) По формуле (используется при подборе пружины и автоёмкости брошюры):

толщина_листа_мм = плотность * 0.0011

Примеры: 130 г/м² → 130 × 0.0011 = 0.143 мм; 80 г/м² → 0.088 мм.

(2) По предустановленной таблице paper_density_thickness (используется при расчёте высоты стопы для гильотины и в карточках материалов). Значение по умолчанию при отсутствии плотности в таблице — 0.15 мм.

Плотность, г/м² Толщина, мм
80 0.10
90 0.11
100 0.12
115 0.14
130 0.16
150 0.18
170 0.20
200 0.24
250 0.30
300 0.36
350 0.42
400 0.48

Формула веса листа:

вес_листа_г = площадь_листа_м2 * плотность

Размеры форматов (для расчёта площади): SRA3 0.32×0.45 м, A4 0.21×0.297 м, A3 0.297×0.42 м, A5 0.148×0.21 м, A6 0.105×0.148 м.

Пример для SRA3 130 г/м²: (0.32 × 0.45) × 130 = 18.72 г на лист; вес 1000 листов ≈ 18.7 кг.

⚠️ ОТКРЫТЫЙ ВОПРОС (единый, сведён сюда из brochures.md и cutting-lamination-services.md): формула и таблица дают несовместимые значения толщины одного листа — для 80 г/м² формула даёт 0.088 мм, таблица 0.10 мм; для 130 г/м² формула даёт 0.143 мм, таблица 0.16 мм. При расчёте корешка пружины и высоты стопы значения расходятся. Нужно выбрать единый источник толщины (формула density × 0.0011 либо таблица paper_density_thickness) и применять его во всех модулях.

8.2. Автоматический подбор пружины#

Логика подбора пружины по количеству листов и плотности бумаги:

sheet_thickness  = paper_density * 0.0011        # толщина одного листа, мм
spine_thickness  = sheets * sheet_thickness      # толщина корешка (блока)
required_thickness = spine_thickness * 1.1       # с запасом (10%)

# Пересчёт лимитов листов пружины под текущую плотность (база 80 г/м²)
ratio        = paper_density / 80
adjusted_min = ceil(min_sheets_80g / ratio)
adjusted_max = floor(max_sheets_80g / ratio)
# Пружина подходит, если adjusted_min <= sheets <= adjusted_max

Предупреждение: при spine_thickness > 8 мм рекомендуется металлическая пружина.

Пример (30 листов, бумага 130 г/м²):

  • толщина корешка: 30 × 0.16 = 4.8 мм;
  • коэффициент пересчёта плотности 80→130: 1.6;
  • эффективное количество листов: 30 / 1.6 = 19 листов (эквивалент 80 г/м²);
  • пересчёт диапазона металлической пружины 8 мм (21–35 листов для 80 г/м²) под 130 г/м² → 13–22 листа (по другому расчёту 14–24).

Тираж 30 листов укладывается в исходный диапазон 80 г/м², пружина 8 мм подбирается.

Альтернативные варианты, которые калькулятор отбраковывает (для тех же 30 листов 130 г/м²):

Пружина Цена, руб Диапазон (80 г/м²), листов Диапазон под 130 г/м², листов Итог
Металлическая 8 мм 18 21–35 13–22 подходит
Пластиковая 10 мм 22 36–50 23–31 не подходит (тираж 30 ниже диапазона)
Металлическая 6 мм 15 10–20 6–12 не подходит (тираж 30 выше диапазона)

⚠️ ОТКРЫТЫЙ ВОПРОС: в примерах подбора пружины используются разные коэффициенты пересчёта плотности: один раз 80→130 даёт коэффициент 1.45 (по толщине 0.16/0.11), другой раз — коэффициент 1.6 (80→130 напрямую как 130/80). Это даёт разные пересчитанные диапазоны (14–24 против 13–22 листов). Нужно унифицировать метод пересчёта.

8.5. Расчёт количества листов и веса заказа (для доставки)#

Это каноническое описание расчёта веса заказа для доставки. Из калькулятора резки/ламинации на него даётся ссылка (см. Резка, ламинация и доп. услуги, раздел 8).

sheets      = ceil(qty / items_per_sheet)
sheetArea   = (width × height) / 1 000 000        # м²
sheetWeight = sheetArea × density                 # граммы
вес_заказа_г = sum( sheetWeight × sheets )         # = площадь_листа_м2 × плотность × sheets
totalKg     = вес_заказа_г / 1000

Справочные размеры листа: A4 210×297, A5 148×210, A6 105×148, SRA3 320×450 (мм).

Пример отображения клиенту: листовки A4 130 г/м², 100 шт → 50 листов SRA3; вес одного листа SRA3 ≈ 18.7 г; общий вес 50 × 18.7 = 935 г ≈ 0.94 кг.

Расчёт стоимости доставки по весу (демо-логика): до 100 кг — бесплатно; свыше — 50 руб за каждые начатые 10 кг.

Дополнительная опция доставки в интерфейсе клиента: срочная доставка в субботу (+500 руб) — отдельный платный чекбокс, не зависящий от веса заказа.

⚠️ ОТКРЫТЫЙ ВОПРОС: в примере вес считается по площади SRA3 (320×450), хотя изделие — листовки А4; при этом порог бесплатной доставки 100 кг для печатной продукции выглядит нереалистично высоким (вероятно, демо-значение). Параметры доставки требуют уточнения.

9. Справочник форматов#

  • Стандартные: A4, A3, SRA3, A5, A6, евро.
  • Нестандартные: произвольные размеры (custom).
  • Спецформаты для C6085: 330×487, 330×762.

Размеры стандартных форматов (мм): A4 210×297, A3 297×420, A5 148×210, A6 105×148, SRA3 320×450.

Учёт вылетов настраивается на уровне конструктора услуги (правила bleed_rules): вылеты могут быть 0 мм (печать на готовый формат, совпадающий с бумагой), 3 мм (резка под стандарт меньшего размера) или 2 мм (нестандартный размер). Само управление вылетами и формулы относятся к разделу конструкторов услуг (admin/constructors) и здесь приводятся только как справочная база форматов.

10. Справочник цветности#

Типы печати (цветность): 1+0, 1+1, 4+0, 4+4.

11. Справочник операций#

  • Печатные: цветность 1+0, 1+1, 4+0, 4+4.
  • Послепечатные: резка, биговка, фальцовка, ламинация.
  • Сборочные: брошюровка на скобу/пружину (в разработке).

12. Что требует доработки#

  • Интерфейс ввода формул — чтобы менеджер мог писать простые выражения без программиста.
  • Валидация зависимостей — проверка совместимости материалов и оборудования.
  • Интеграция с производственным календарём — загрузка оборудования.
  • Обратная связь от производства — фактическое время vs расчётное.
