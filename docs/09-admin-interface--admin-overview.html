Админка: обзор

Административная панель PrintOffice — это модуль настроек, позволяющий управлять всей бизнес-логикой типографии (конструкторами услуг, ценами, формулами, справочниками, ролями) без участия программиста. Все изменения вносятся через визуальные конструкторы и справочники.

Назначение админки

Админ-панель служит единым центром управления конфигурацией системы. Через неё настраиваются:

  • страницы-калькуляторы (конструкторы услуг);
  • ценообразование и маржа;
  • расчётные формулы и правила валидации;
  • справочники оборудования, материалов, форматов, услуг и партнёров;
  • пользователи и права доступа.

Сквозной принцип: «настройка без программиста»

Ключевой принцип админ-панели — администратор должен иметь возможность менять поведение системы самостоятельно, без привлечения разработчиков. Это обеспечивается за счёт следующих механизмов:

  • Конструкторы — сборка калькуляторов и страниц из готовых блоков.
  • Визуальный редактор формул — настройка расчётных формул без написания кода, с проверкой синтаксиса и тестовым расчётом.
  • Конструктор правил валидации — создание правил совместимости (материалов, форматов) без программиста.
  • Справочники — ведение оборудования, материалов, форматов, услуг и партнёров через формы.
  • Шаблоны ответов — заготовки текстов для коммуникаций.
  • Импорт/экспорт Excel — массовая загрузка и выгрузка цен партнёров.

Разделы настроек

Модуль настроек разделён на функциональные блоки.

1. Конструкторы услуг

Сборка калькуляторов из блоков. Каждый конструктор привязан к отдельной странице сайта (каждый конструктор = своя страница).

Возможности:

  • Создание страниц калькуляторов.
  • Привязка к страницам сайта.
  • Настройка доступных материалов (какие бумаги показывать).
  • Настройка доступных форматов (какие форматы доступны).
  • Настройка параметров (форматы, бумага, цветность).
  • Правила вылетов для конструктора (гибкие условия).
  • Технологические операции по умолчанию (что включено в услугу).
  • Привязка дополнительных услуг.
  • Копирование конструкторов (быстрое создание новых страниц).
  • Семейства товаров — группировка вариантов (скоба / пружина / КБС).
  • Кросс-продажи — настройка связей между услугами.

2. Маржа и цены

Управление ценообразованием.

  • Таблицы маржи (по тиражу, по себестоимости).
  • Импорт/экспорт Excel для цен партнёров.
  • Настройка налога (7%).
  • Правила доставки (регионы, вес, тираж, время).

3. Формулы и правила (выбор технологии)

Настройка расчётной логики и правил совместимости.

Конструктор формул:

  • Визуальный редактор.
  • Доступные переменные.
  • Проверка синтаксиса.
  • Тестовый расчёт.
  • История версий.
  • Шаблоны формул.

Конструктор правил валидации:

  • Условия совместимости (материалов, форматов).
  • Уровни серьёзности: ошибка / предупреждение.
  • Сообщения и рекомендации.
  • Исключения для постоянных клиентов.

Прочее:

  • Шаблоны ответов.

4. Справочники

Ведение базовых сущностей системы. Создание/редактирование оборудования, материалов и форматов.

Справочник Состав / примеры Примечание
Оборудование (принтеры) принтеры, резаки, плоттеры, ламинаторы с техническими полями и скоростями
Материалы бумага, плёнки, пружины, картон, самоклейка бумага, плёнки
Форматы A4, A5, A6, A3, SRA3, нестандартные стандартные и нестандартные
Услуги (операции) ламинация, биговка, фальцовка, сверление технологические операции
Партнёры ГМ Принт и др. —

Дополнительно к справочникам относятся:

  • Медиа-галерея — загрузка фото и видео для материалов.
  • Шаблоны для менеджера — быстрый выбор формата.
  • Шаблоны ответов — для коммуникаций.

5. Витрина

Настройка отображения услуг на сайте через конструкторы услуг (привязка к страницам, выбор показываемых материалов и форматов, кросс-продажи, семейства товаров). См. раздел «Конструкторы услуг».

6. Пользователи и роли

Управление доступом сотрудников и клиентов.

  • Управление ролями: admin, manager, partner, client.
  • Права доступа.
  • Приглашение сотрудников.

7. Брифы, уведомления, шаблоны документов

⚠️ ОТКРЫТЫЙ ВОПРОС: в указанных фрагментах источника явно не описаны разделы настройки брифов, уведомлений и шаблонов документов как отдельные блоки админки. В исходнике присутствуют только «шаблоны ответов» (для коммуникаций) и «шаблоны для менеджера» (быстрый выбор формата). Требуется уточнить состав и наличие этих разделов.

Кто имеет доступ

Доступ к админ-панели определяется ролями пользователей:

Роль Доступ
admin полный доступ к настройкам, справочникам, ролям и формулам
manager работа в рамках предоставленных прав (например, шаблоны для менеджера)
partner доступ к данным партнёра (цены через импорт/экспорт Excel)
client клиентский доступ (без настроек админки)

⚠️ ОТКРЫТЫЙ ВОПРОС: точное распределение прав доступа по разделам админки между ролями admin, manager, partner, client в источнике детально не описано — указан только сам перечень ролей и наличие механизма прав доступа.
