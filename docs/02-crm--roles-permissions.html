Роли и права

Описание системы ролей CRM типографии PrintOffice: перечень предопределённых ролей, матрица доступа «роль → раздел → действие», правила доступа к файлам в зависимости от типа клиента и типа заказа, а также механизм настройки прав силами администратора без участия программиста.

Список ролей

Администратор создаёт роли и назначает разрешения для каждого раздела. Базовый набор ролей:

  • Менеджер по продажам — работа с заказами и клиентами, коммуникации с клиентами, создание заказов, изменение статусов заказа (кроме статуса «Завершён»). Может запускать доставку.
  • Дизайнер — доступ к файлам (папки заказов, назначенных на него), внутренние комментарии.
  • Технолог — производство (вкладка в карточке заказа), комментарии для цеха, просмотр файлов (готовые макеты).
  • Отгрузчик — отгрузка: список заказов со статусом «Готов к отгрузке», печать накладных, вызов курьера, смена статуса на «Отгружен».
  • Бухгалтер — финансы, документы, синхронизация с Эльбой.
  • Руководитель — полный доступ ко всем разделам, включая настройки.

Матрица «роль → раздел → действие»

Права настраиваются по принципу: для каждой роли указывается набор разделов и допустимых действий (читать / писать / удалять). Сводная матрица доступа по ролям:

Уровни действий для каждой пары «роль → раздел» задаются в терминах: читать / писать / удалять.

Ограничения по статусам заказа

  • Менеджер по продажам может менять статусы заказа, кроме перевода в статус «Завершён».
  • Отгрузчик работает только с заказами в статусе «Готов к отгрузке» и переводит их в статус «Отгружен».

Доступ к файлам по ролям и статусу заказа

Видимость и операции с файлами определяются ролью пользователя:

  • Дизайнер — доступ к папкам только тех заказов, которые назначены на него; может оставлять внутренние комментарии.
  • Технолог — просмотр готовых макетов по заказам.
  • Менеджер — управление файлами в карточке заказа (загрузка, скачивание, предпросмотр, перемещение в архив, удаление в корзину).
  • Доступ к разделу «Файлы» (интеграция с Яндекс.Диском, просмотр всех папок заказов) предоставляется по правам роли.

Категории и сроки хранения файлов

Категория хранения и срок жизни файла (стратегия FileStorageStrategy: temp 24/72 ч, active 30/90 дн, archive бессрочно) определяются типом заказа, типом клиента и ролью файла, а не ролью пользователя. Эти параметры не зависят от прав доступа и описаны отдельно — каноническое описание класса FileStorageStrategy и таблиц сроков см. в Управление файлами и макетами (раздел «Стратегия хранения в зависимости от типа клиента»).

Настройка прав без программиста

  • Создание ролей и назначение разрешений по каждому разделу выполняет администратор через интерфейс.
  • Все роли и права настраиваются через интерфейс без участия программиста на основе таблиц вида «роль → раздел → действие (читать / писать / удалять)».
  • Раздел «Настройки» содержит справочники (партнёры, офисы, роли, правила бота, маржа); доступ к нему имеют роли с соответствующими правами (Руководитель — полностью).
