Обзор продукта

Обзорный документ системы управления типографией PrintOffice: назначение продукта, целевая аудитория, решаемые задачи, масштаб бизнеса, цели и ключевые принципы построения системы.

Что за продукт#

PrintOffice — полноценная система управления типографией (CRM + калькулятор заказов + производственный учёт + сайт с автоматическим расчётом). Система проектируется для типографии с тремя офисами в Москве и развёртывается на собственном сервере (домен https://printoffice.moscow).

Система покрывает весь путь заказа: от первого контакта клиента до выдачи готовой продукции и закрывающих документов. Она:

  • Автоматизирует расчёты — от простых листовок до сложных брошюр с разными блоками.
  • Учитывает реальное производство — раскладка макетов на печатном листе, вылеты, поля принтера, метки плоттера.
  • Поддерживает офсетных и сторонних партнёров — импорт цен из Excel/с сайтов партнёров, расчёт маржи, учёт доставки и налогов.
  • Планирует загрузку — календари оборудования, приоритеты, сроки.
  • Собирает обратную связь — самообучение, корректировка нормативов по фактическим данным.
  • Настраивается без программиста — конструкторы услуг, формулы, маржа, справочники.
  • Увеличивает средний чек — кросс-продажи, рекомендации, комплекты.

Для кого и какую боль решает#

Продукт предназначен для типографии, которая часть продукции выпускает сама, а часть (широкоформатная, офсетная, сувенирная печать) перезаказывает у подрядчиков.

Пользователи системы:

  • Руководитель (владелец) — контроль, аналитика, история.
  • Менеджеры (2 человека) — приём заявок, расчёты, согласования, планирование производства.
  • Клиенты — оформление и расчёт заказов на сайте, повторные заказы, получение документов.
  • Партнёры/подрядчики — взаимодействие через портал (например, для Эдгара) или через парсинг входящих писем (ГМ Принт).

Болевые точки, которые решает система (формулировка целей):

  • Автоматизировать расчёт заказов через сайт, чтобы клиенты сами оформляли заказы, указывая все параметры, и освобождали менеджера от рутинного расчёта стандартных заказов; менеджер подключается, только когда возможностей сайта не хватает для полного расчёта и ответов на вопросы.
  • Автоматически формировать и выдавать клиентам закрывающие документы.
  • Обеспечить лёгкий поиск и повторение прошлых заказов.
  • Не терять макеты и заказы, не искать их при повторном обращении.
  • Напоминать клиентам о компании, чтобы удерживать их.
  • Создать собственную базу цен на широкоформатную печать, чтобы не запрашивать стоимость у партнёра каждый раз, а сразу делать наценку и называть сроки.

Масштаб бизнеса#

Показатель Значение
Среднее количество заказов в день 20
Пиковое количество заказов в день (сезон) 30–35
Средний чек 5 000 ₽
Минимальный чек 1 000 ₽
Максимальный чек 250 000 ₽
Уникальных активных клиентов в месяц 100
Количество менеджеров 2
Активные подрядчики GMPrint, Wideformat, ФастПринт, ПринтХит
Количество услуг/позиций в прайсе ~100
Максимальный размер макета до 1500 МБ

Распределение клиентов по типу:

Тип клиента Доля
Физлица (B2C) 20 %
Юрлица (B2B) 80 %

Дополнительные параметры бизнеса:

  • Документы для юрлиц: счёт / УПД / счёт-договор. С частью юрлиц подписываются договоры электронно по ЭДО (Диадок).
  • НДС: на текущий момент компания работает без НДС, планируется переход на работу с НДС.
  • Налоговая ставка, учитываемая в наценке при перезаказе у подрядчиков, — 7 %.
  • Собственной широкоформатной, офсетной и сувенирной печати нет — такая продукция перезаказывается у подрядчиков (с учётом наценки за вычетом расходов на доставку до офиса и налогов).
  • У офсетных партнёров (3 партнёра) цены и сроки берутся с их сайтов; на эту базу делается своя наценка с учётом расходов на налоги и доставку.

Цели продукта#

  • Перенести расчёт стандартных заказов на сайт (самообслуживание клиентов), снизив нагрузку на менеджеров.
  • Централизованно хранить заказы, макеты и историю коммуникаций; обеспечить быстрый повтор заказов.
  • Автоматизировать формирование и отправку закрывающих документов (по ЭДО Диадок).
  • Вести базу цен партнёров (в т. ч. для широкоформатной печати) для мгновенного расчёта с наценкой.
  • Планировать производство с учётом загрузки оборудования, наличия материалов и сроков.
  • Удерживать клиентов через напоминания и систему лояльности.

Ключевые особенности и принципы построения системы#

Особенность / принцип Описание
Гибкость / настройка без программиста Всё настраивается в админке: конструкторы, формулы, маржа, справочники.
Все сущности имеют ID Любой объект в системе имеет уникальный идентификатор.
Конструкторы из блоков Любая услуга собирается из готовых блоков (печать, материалы, сборка).
Единый механизм маржи Маржа настраивается единообразно для любых услуг.
Гибкие формулы Формулы настраиваются в админке без участия программиста.
Самообучение / точность Система корректирует нормативы на основе фактической обратной связи с производства; самообучающиеся формулы на реальных данных.
Мультипартнёрность Поддержка нескольких офсетных партнёров с разными ценами и сроками.
Разделение B2C/B2B Разные интерфейсы для частных и корпоративных клиентов.
Прозрачность Полная цепочка от расчёта до готового заказа.
Скорость Автоматизация рутинных операций.
Контроль Полная история и аналитика.
Масштабирование Лёгкое добавление нового оборудования.
Учёт реального производства Раскладка, вылеты, поля принтера, метки плоттера.
Увеличение чека Кросс-продажи, рекомендации, комплекты.
