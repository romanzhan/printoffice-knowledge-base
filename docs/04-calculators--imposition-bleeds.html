Раскладка на листе и вылеты

Спецификация алгоритма раскладки макетов на печатном листе (impositions) и правил вылетов (bleeds) для калькулятора типографии: расчёт количества изделий на лист с учётом обеих ориентаций, технологических полей принтера, меток плоттера для контурной резки и выбора печатного формата.

Базовые принципы раскладки#

Раскладка макетов на печатном листе всегда строится по следующим правилам:

  1. К размеру изделия добавляются вылеты (bleed) с каждой стороны.
  2. Проверяются обе ориентации макета (книжная и альбомная) — система ищет, в какой ориентации помещается больше изделий.
  3. Выбирается вариант с максимальным количеством изделий на листе (с учётом ограничений по полям и технологичности).
  4. Из печатного листа вычитаются технологические поля принтера (непечатаемая область) и, при контурной резке, место под метки плоттера и безопасный отступ.
  5. Менеджеру оставляется возможность вручную скорректировать раскладку и параметры для сложных заказов.

Базовая формула количества листов на тираж:

items_per_sheet = cols × rows           // максимум по обеим ориентациям
sheets = ceil(qty / items_per_sheet)

Расчёт раскладки A6 на SRA3#

Размеры: SRA3 = 320 × 450 мм, A6 = 105 × 148 мм.

Геометрический максимум (без полей)#

По ширине:  320 ÷ 105 = 3.04 → 3 шт   (остаток 320 - 315 = 5 мм)
По высоте:  450 ÷ 148 = 3.04 → 3 шт   (остаток 450 - 444 = 6 мм)
ИТОГО: 3 × 3 = 9 шт на листе (максимальная плотность, требует точной резки)

Проверка с поворотом по ширине: 320 ÷ 148 = 2.16 → 2 шт — хуже, поэтому при печати «в плотную» оставляют 3 по ширине.

Раскладка с вылетами 3 мм (рабочий вариант)#

Размер A6 с вылетами 3 мм с каждой стороны:

Ширина с вылетами = 105 + 3 + 3 = 111 мм
Высота с вылетами = 148 + 3 + 3 = 154 мм

Проверка обеих ориентаций на SRA3:

Ориентация Размер макета По ширине По высоте Итого
Книжная 111 × 154 320 ÷ 111 = 2.88 → 2 450 ÷ 154 = 2.92 → 2 4 шт
Альбомная 154 × 111 320 ÷ 154 = 2.07 → 2 450 ÷ 111 = 4.05 → 4 8 шт

Оптимальная раскладка: альбомная ориентация, 2 × 4 = 8 шт на листе.

Альбомная (154 × 111):
По ширине:  320 ÷ 154 = 2.07 → 2 шт   (остаток 320 - 308 = 12 мм на поля)
По высоте:  450 ÷ 111 = 4.05 → 4 шт   (остаток 450 - 444 = 6 мм на поля)
ИТОГО: 2 × 4 = 8 шт

Варианты раскладки A6 на SRA3 (как опции)#

const layouts = {
  dense_9:    { cols: 3, rows: 3, rotated: false, description: "Максимальная плотность (9 шт, без полей)", requires: "точная резка" },
  rotated_8:  { cols: 2, rows: 4, rotated: true,  description: "С поворотом (8 шт)",                       requires: "стандартные поля" },
  technical_6:{ cols: 3, rows: 2, rotated: false, description: "С технологическими полями (6 шт)",         requires: "надёжная резка" }
};

⚠️ ОТКРЫТЫЙ ВОПРОС: для варианта rotated_8 в одном месте источника указаны cols: 4, rows: 2, а в выверенном расчёте — cols: 2, rows: 4 (альбомная: 2 по ширине × 4 по высоте). Геометрически верен вариант 2 × 4. Также не зафиксировано, какая раскладка A6 (6, 8 или 9 шт) реально используется в производстве по умолчанию — вопрос «6 или 8?» оставался без ответа.

Пример пересчёта тиража (A6, 100 шт, раскладка 8 шт/лист)#

Печатных листов = 100 ÷ 8 = 12.5 → 13 листов SRA3
Бумага:  13 × 3  = 39 руб
Печать:  13 × 22 = 286 руб
Себестоимость = 325 руб
Цена клиенту (наценка 100%): 325 × 2 = 650 руб → 6.50 руб/шт

Сравнение: раскладка 6 шт/лист дала бы ceil(100/6) = 17 листов и себестоимость 425 руб; раскладка 8 шт/лист — 13 листов и 325 руб. Экономия ~100 руб себестоимости / 200 руб в цене клиента при сохранении прибыли.

Алгоритм раскладки (итоговая версия для калькулятора)#

function calculateLayout(printFormat, productFormat, bleed, isDoubleSided) {
  // printFormat = { width: 320, height: 450 } // SRA3
  // productFormat = { width: 105, height: 148 } // A6
  // bleed = 3 // мм

  const itemWidth  = productFormat.width  + (bleed * 2);
  const itemHeight = productFormat.height + (bleed * 2);

  const orientations = [
    { width: itemWidth,  height: itemHeight, name: "книжная" },
    { width: itemHeight, height: itemWidth,  name: "альбомная" }
  ];

  let bestLayout = { count: 0, cols: 0, rows: 0, rotated: false };

  for (const orient of orientations) {
    // -10 мм на захват и поля по краям листа
    const cols = Math.floor((printFormat.width  - 10) / orient.width);
    const rows = Math.floor((printFormat.height - 10) / orient.height);
    const total = cols * rows;

    if (total > bestLayout.count) {
      bestLayout = {
        count: total, cols, rows,
        rotated: (orient.name === "альбомная"),
        orientation: orient.name
      };
    }
  }
  return bestLayout;
}
// A6 на SRA3 с вылетами 3 мм → { count: 8, cols: 2, rows: 4, rotated: true, orientation: "альбомная" }

Расчёт нестандартных размеров#

Для произвольного размера система ищет оптимальное размещение на стандартных печатных листах (SRA3, A3, A4) с учётом вылетов, технологических полей и обеих ориентаций.

Алгоритм#

  1. Добавить вылеты к размеру макета: itemW = width + bleed*2, itemH = height + bleed*2.
  2. Перебрать все доступные печатные форматы принтера.
  3. Для каждого формата проверить обе ориентации.
  4. Рассчитать остатки полей и проверить достаточность (минимально допустимое поле MIN_MARGIN = 5 мм).
  5. Отсортировать варианты: сначала безопасные (isSafe), затем по убыванию количества.
  6. Предложить менеджеру лучшие варианты с возможностью ручной корректировки.

Пример: макет 90 × 50 мм, вылеты 3 мм → 96 × 56 мм#

Формат Ориентация Размер макета Расчёт Кол-во Остатки Приоритет
SRA3 книжная 96 × 56 320÷96=3, 450÷56=8 24 по высоте 2 мм ⚠️ риск
SRA3 альбомная 56 × 96 320÷56=5, 450÷96=4 20 хорошие (40/66 мм) ✅
A3 книжная 96 × 56 297÷96=3, 420÷56=7 21 средние (9/28 мм) ✅
A3 альбомная 56 × 96 297÷56=5, 420÷96=4 20 хорошие (17/36 мм) ✅
A4 книжная 96 × 56 210÷96=2, 297÷56=5 10 хорошие ❌ мало
A4 альбомная 56 × 96 210÷56=3, 297÷96=3 9 хорошие ❌ мало

Для тиража 1000 шт нестандарта 90 × 50 при раскладке 24 шт/лист SRA3: 1000 ÷ 24 = 41.66 → 42 листа.

Код алгоритма#

function calculateNonStandardLayout(productWidth, productHeight, bleed = 3) {
  const printFormats = [
    { name: "SRA3", width: 320, height: 450, costMultiplier: 1.0 },
    { name: "A3",   width: 297, height: 420, costMultiplier: 0.9 },
    { name: "A4",   width: 210, height: 297, costMultiplier: 0.7 }
  ];

  const itemW = productWidth  + (bleed * 2);
  const itemH = productHeight + (bleed * 2);
  const results = [];
  const MIN_MARGIN = 5; // минимально допустимое поле для захвата и резки

  for (const format of printFormats) {
    const orientations = [
      { w: itemW, h: itemH, name: "книжная" },
      { w: itemH, h: itemW, name: "альбомная" }
    ];

    for (const orient of orientations) {
      const cols = Math.floor((format.width  - MIN_MARGIN * 2) / orient.w);
      if (cols < 1) continue;
      const rows = Math.floor((format.height - MIN_MARGIN * 2) / orient.h);
      if (rows < 1) continue;

      const usedWidth  = cols * orient.w;
      const usedHeight = rows * orient.h;
      const marginLeft = Math.floor((format.width  - usedWidth)  / 2);
      const marginTop  = Math.floor((format.height - usedHeight) / 2);
      const marginRight  = format.width  - usedWidth  - marginLeft;
      const marginBottom = format.height - usedHeight - marginTop;
      const minMargin = Math.min(marginLeft, marginRight, marginTop, marginBottom);

      results.push({
        format: format.name, orientation: orient.name,
        cols, rows, total: cols * rows,
        usedWidth, usedHeight,
        margins: { left: marginLeft, right: marginRight, top: marginTop, bottom: marginBottom, min: minMargin },
        isSafe: minMargin >= MIN_MARGIN,
        warning: minMargin < MIN_MARGIN ? `Малое поле: ${minMargin} мм` : null
      });
    }
  }

  results.sort((a, b) => {
    if (a.isSafe && !b.isSafe) return -1;
    if (!a.isSafe && b.isSafe) return 1;
    return b.total - a.total;
  });
  return results;
}

Особые случаи нестандартной раскладки#

  • Макет чуть-чуть не влезает (на 1–2 мм): предложить уменьшить вылеты до 2 мм, предупредить о рисках при резке.
  • Много разных макетов в одном заказе: раскладка считается отдельно для каждого макета; применяется коэффициент сложности; менеджер видит предупреждение.
  • Двусторонняя печать: ориентация должна быть одинаковой на обеих сторонах; неподходящие варианты раскладки система отфильтровывает автоматически.

Правила вылетов (bleed)#

Производственные правила:

  • Стандартные форматы: вылеты 3 мм с каждой стороны для резки.
  • Нестандартные форматы: допускается уменьшить вылеты до 2 мм с каждой стороны, если позволяет макет (на усмотрение менеджера). Чаще применяется на односторонней печати, но возможно и на двусторонней, если выгоднее.

Итоговая матрица правил вылетов#

Категория Вылеты Условие
Стандартный формат (A4, A3, A5 и т.д.) 0 Печать на тот же формат, без послепечатной резки
Стандартный формат 3 Печать на большем формате с резкой под стандарт
Нестандартный (произвольный) размер 3 Все случаи (по умолчанию)

Особые условия:

  • При расчёте всегда проверяются обе ориентации макета.
  • При равном количестве изделий в разных ориентациях выбирается вариант с вылетами 3 мм (для технологичности резки).
  • Если вариант с вылетами 0 мм даёт больше изделий, но формат нестандартный, он не используется — только 3 мм.

⚠️ ОТКРЫТЫЙ ВОПРОС: в производственном описании заказчик допускает для нестандарта вылеты 2 мм (по согласованию с менеджером), но формализованная матрица правил (BLEED_RULES) фиксирует для нестандарта жёстко 3 мм во всех случаях. Не определено, как в калькуляторе совмещаются «всегда 3 мм для нестандарта» и опция ручного уменьшения до 2 мм.

Правила вылетов в коде#

const BLEED_RULES = {
  default: 3,
  conditions: [
    { name: "Стандартный формат, печать без резки", condition: "is_standard && matches_paper",  bleed: 0 },
    { name: "Стандартный формат с резкой",          condition: "is_standard && !matches_paper", bleed: 3 },
    { name: "Нестандартный размер",                 condition: "!is_standard",                  bleed: 3 }
  ],
  tieBreaker: {
    rule: "При равном количестве элементов в разных ориентациях",
    action: "выбирается вариант с вылетами 3 мм",
    reason: "Обеспечение технологичности резки"
  }
};

Тест равного количества (нестандарт 80 × 120 мм на SRA3)#

С вылетами 3 мм (86 × 126):
  Книжная:   320÷86=3,  450÷126=3 → 9 шт
  Альбомная: 320÷126=2, 450÷86=5  → 10 шт ✅
С вылетами 0 мм (80 × 120):
  Книжная:   320÷80=4,  450÷120=3 → 12 шт
  Альбомная: 320÷120=2, 450÷80=5  → 10 шт
Результат: лучший по количеству = 12 шт (0 мм), но для нестандарта 0 мм недопустимо
           → выбираем следующий лучший с 3 мм = 10 шт.

Примеры применения правил вылетов#

Ситуация Формат Стандарт? Совпадает с бумагой? Вылеты Результат
Визитки 90×50 90×50 Нет — 3 мм Всегда 3 мм для нестандарта
Листовки A5 148×210 Да Нет (печать на SRA3) 3 мм Стандарт с резкой
Документы A4 210×297 Да Да (печать на A4) 0 мм Готовый формат
Конверт C5 162×229 Да Нет (печать на SRA3) 3 мм Стандарт с резкой
Плакат A3 297×420 Да Да (печать на A3) 0 мм Готовый формат
Нестандарт 95×55 95×55 Нет — 3 мм Всегда 3 мм

Тест-кейсы:

  • Нестандарт 90×50, SRA3: с 3 мм — 20 шт (альбомная); с 0 мм — 24 шт (недопустимо) → 20 шт с 3 мм.
  • A5 на SRA3 (стандарт с резкой): с 3 мм — 8 шт; с 0 мм — 12 шт (нельзя, нужна резка) → 8 шт с 3 мм.
  • A4 на A4 (стандарт без резки): и 3 мм, и 0 мм дают 1 шт → 1 шт с 0 мм (выбирается более технологичный).

Технологические поля принтера (непечатаемая область)#

Каждый принтер имеет непечатаемую область — поля по краям листа, где изображение физически не пропечатается. Эти поля настраиваются в админке для каждого принтера и формата и вычитаются при расчёте раскладки.

Поля Konica Minolta C6085#

Формат листа: 320 × 450 мм (SRA3)
Технологические поля (непечатаемые):
  Сверху:  3 мм
  Снизу:   3 мм
  Слева:   1 мм
  Справа:  1 мм
Реальная печатная область: 314 × 444 мм  (320 - 1 - 1 = 318 по ширине см. ниже)

Формула печатной области:

printable_width  = sheet_width  - left_margin - right_margin
printable_height = sheet_height - top_margin  - bottom_margin

⚠️ ОТКРЫТЫЙ ВОПРОС: в источнике печатная область C6085 указана как 314 × 444 мм (через расчёт «320-6, 450-6», то есть −3/−3 по обеим осям), но при полях 1/1 слева/справа и 3/3 сверху/снизу ширина должна быть 320 − 1 − 1 = 318 мм, а высота 450 − 3 − 3 = 444 мм (то есть 318 × 444). Значения 314 и 318 по ширине противоречат друг другу в разных местах текста.

Схема БД технологических полей#

CREATE TABLE printer_technical_specs (
  id INT PRIMARY KEY AUTO_INCREMENT,
  printer_id INT NOT NULL,
  printable_width INT,            -- мм
  printable_height INT,
  top_margin INT,                 -- сверху (обычно 3-5 мм)
  bottom_margin INT,
  left_margin INT,
  right_margin INT,
  margins_symmetric BOOLEAN DEFAULT true,
  max_paper_weight INT,           -- макс. плотность бумаги
  max_sheets_per_run INT,
  valid_from DATE, valid_to DATE, is_active BOOLEAN DEFAULT true,
  FOREIGN KEY (printer_id) REFERENCES printers(id),
  INDEX idx_printer_active (printer_id, is_active)
);

Учёт полей в раскладке#

const availableWidth  = printFormat.width  - left_margin - right_margin;
const availableHeight = printFormat.height - top_margin  - bottom_margin;
// далее раскладка считается только внутри (availableWidth × availableHeight)
const cols = Math.floor(availableWidth  / itemWidth);
const rows = Math.floor(availableHeight / itemHeight);

Метки плоттера для контурной резки#

При контурной резке наклеек или вырубке на режущем плоттере на лист добавляются угловые метки, по которым плоттер определяет координаты элементов. Метки занимают полезную площадь и уменьшают область раскладки.

Ключевые параметры#

  • Смещение меток от края листа: обычно 8 мм (настраивается, рабочий диапазон 5–10 мм).
  • Безопасный отступ макетов от меток: 10 мм по горизонтали и вертикали — макет не должен налезать на метки или касаться их.

Важное уточнение: метки не вычитаются из печатной области дважды#

Метки на расстоянии 8 мм от края попадают в печатаемую область (так как 8 мм > максимального технологического поля 3 мм), поэтому они корректно печатаются. Непечатаемая область влияет только на то, что изображение в ней не пропечатается, но размещать элементы (включая метки) в этой зоне не запрещено. Метки не уменьшают печатную область — они задают рамку, внутри которой размещаются макеты.

Проверка попадания меток в печатаемую область:
  markerOffset >= top_margin   →  8 >= 3 ✓
  markerOffset >= left_margin  →  8 >= 1 ✓

Итоговая формула области для макетов#

Область для макетов считается с учётом двух вычетов: смещения меток от края и безопасного отступа от меток.

item_area_width  = sheet_width  - (markerOffset × 2) - (safetyGap × 2)
item_area_height = sheet_height - (markerOffset × 2) - (safetyGap × 2)
startX = markerOffset + safetyGap
startY = markerOffset + safetyGap

Для SRA3 (320 × 450), markerOffset = 8 мм, safetyGap = 10 мм:

Область внутри меток:  320 - 16 = 304 мм ширина;  450 - 16 = 434 мм высота   (304 × 434)
Область для макетов:   304 - 20 = 284 мм ширина;  434 - 20 = 414 мм высота   (284 × 414)
Начало координат: (18, 18)

function calculateLayoutArea(paperWidth, paperHeight, markerOffset, safetyGap) {
  const areaInsideMarkers = {
    width:  paperWidth  - (markerOffset * 2),
    height: paperHeight - (markerOffset * 2)
  };
  const areaForItems = {
    width:  areaInsideMarkers.width  - (safetyGap * 2),
    height: areaInsideMarkers.height - (safetyGap * 2)
  };
  return {
    total: areaInsideMarkers,        // 304 × 434
    forItems: areaForItems,          // 284 × 414
    // строковая формула для отображения в UI: "(320 - 8×2 - 10×2) = 284"
    formula: `(${paperWidth} - ${markerOffset}×2 - ${safetyGap}×2) = ${areaForItems.width}`,
    startX: markerOffset + safetyGap, // 18
    startY: markerOffset + safetyGap  // 18
  };
}
// calculateLayoutArea(320, 450, 8, 10) → forItems { width: 284, height: 414 }

Зависимость области для макетов от смещения меток (SRA3, safetyGap отдельно)#

Таблица показывает область внутри меток (304 = 320 − 8×2 и т.д.) — без вычета безопасного отступа:

Смещение меток, мм Ширина внутри меток Высота внутри меток Начало X Начало Y Примечание
3 314 444 3 3 Метки на границе поля
5 310 440 5 5 ✅ рабочий вариант
8 304 434 8 8 ✅ стандарт
10 300 430 10 10 ✅ большой запас
15 290 420 15 15 ❌ много места теряем

Менеджер может вручную менять смещение меток (например, на 5 мм), чтобы разместить больше элементов; область пересчитывается автоматически по формуле выше.

Пример раскладки наклеек с контурной резкой#

Наклейка 70 × 50 мм, вылеты 3 мм → 76 × 56 мм. Область для макетов на SRA3 с метками 8 мм и отступом 10 мм = 284 × 414 мм.

Книжная   (76 × 56):  284÷76=3, 414÷56=7  → 21 шт
Альбомная (56 × 76):  284÷56=5, 414÷76=5  → 25 шт ✅

Оптимум — альбомная, 25 шт/лист. На тираж 500 шт: 500 ÷ 25 = 20 листов.

Проверочный пример себестоимости (тираж 500, C6085, контурная резка):

Печать: 20 × 22 руб = 440 руб
Бумага: 20 × 3 руб  = 60 руб
Резка:  500 × 0.2 мин × 30 руб/мин = 3000 руб
ИТОГО: 3500 руб → 7 руб/наклейка

⚠️ ОТКРЫТЫЙ ВОПРОС: в источнике для одного и того же примера (наклейки 70×50, SRA3) встречаются разные результаты раскладки — 35 шт (ранний ошибочный расчёт без отступа от меток), 25 шт (альбомная, область 284×414) и упоминание «5×7 = 35 шт» при метках без безопасного отступа. Канонический результат с учётом markerOffset 8 + safetyGap 10 = 25 шт/лист (20 листов на 500).

Сравнение раскладок с метками и без (наклейки 50×50 мм, размер с вылетами 56×56)#

Параметр Без контурной резки С контурной резкой
Доступная ширина 320 − 1 − 1 = 318 (с метками/отступом меньше)
Доступная высота 450 − 3 − 3 = 444 (с метками/отступом меньше)
По ширине 318 ÷ 56 = 5 шт 5 шт
По высоте 444 ÷ 56 = 7 шт 7 шт
Итого на листе 35 шт 35 шт (в данном случае совпало)
Себестоимость 1 шт ниже выше (доп. операция резки)

Приоритет расчёта при контурной резке:

  1. Определить поля принтера → печатная область.
  2. Если есть контурная резка → вычесть место под метки и безопасный отступ.
  3. Рассчитать раскладку в итоговой области для макетов (обе ориентации).
  4. Вернуть результат с учётом всех ограничений.

Архитектура форматов бумаги и выбор печатного формата#

Принтер (например, Konica Minolta AccurioPress C6085) может печатать не только на стандартных форматах, но и на нестандартных загрузках (банерный формат 330 × 762 мм, спец. лоток 330 × 487 мм). Печатные форматы и их печатные области хранятся как шаблоны и привязываются к принтерам.

Параметры форматов Konica C6085#

Лоток / формат Размер листа, мм Печатная область, мм Поля (в/н/л/п), мм
SRA3 основной лоток 320 × 450 314 × 444 3 / 3 / 1 / 1
Спец. лоток 330×487 (физ. 330,2 × 487,7) 330 × 487 321 × 480 3 / 3 / 4 / 4
Банерный лоток 330 × 762 321 × 480 3 / 3 / 4 / 4

Примечание: у банерного формата максимальная высота печати ограничена 480 мм (та же печатная область, что у спец. лотка), несмотря на длину листа 762 мм.

SRA3 320 × 450 314 × 444 5 × 4 = 20 25 база

Схема БД форматов и привязки к принтерам#

CREATE TABLE paper_formats (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(255) NOT NULL,                          -- "SRA3", "Банер 330×762"
  category ENUM('standard','custom','roll') DEFAULT 'standard',
  sheet_width INT NOT NULL,
  sheet_height INT NOT NULL,
  is_roll BOOLEAN DEFAULT false,
  roll_length INT,                                     -- длина рулона, м
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_category (category),
  INDEX idx_size (sheet_width, sheet_height)
);

CREATE TABLE printer_paper_formats (
  id INT PRIMARY KEY AUTO_INCREMENT,
  printer_id INT NOT NULL,
  paper_format_id INT NOT NULL,
  local_name VARCHAR(255),                             -- "Основной лоток", "Банерный лоток"
  printable_width INT NOT NULL,
  printable_height INT NOT NULL,
  top_margin INT DEFAULT 3, bottom_margin INT DEFAULT 3,
  left_margin INT DEFAULT 1, right_margin INT DEFAULT 1,
  max_sheets_per_run INT, min_quantity INT DEFAULT 1, setup_time INT,
  is_default BOOLEAN DEFAULT false, is_active BOOLEAN DEFAULT true,
  priority INT DEFAULT 100,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (printer_id) REFERENCES printers(id),
  FOREIGN KEY (paper_format_id) REFERENCES paper_formats(id),
  UNIQUE KEY unique_printer_format (printer_id, paper_format_id),
  INDEX idx_printer_active (printer_id, is_active)
);

CREATE TABLE manager_paper_templates (
  id INT PRIMARY KEY AUTO_INCREMENT,
  printer_id INT NOT NULL,
  template_name VARCHAR(255) NOT NULL,                 -- видит менеджер
  paper_format_id INT NOT NULL,
  default_bleed INT DEFAULT 3,
  default_markup_percent DECIMAL(5,2),
  service_types JSON,                                  -- ['envelopes','posters','stickers']
  description TEXT,
  sort_order INT DEFAULT 0,
  is_active BOOLEAN DEFAULT true,
  created_by INT, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (printer_id) REFERENCES printers(id),
  FOREIGN KEY (paper_format_id) REFERENCES paper_formats(id),
  INDEX idx_printer_sort (printer_id, sort_order)
);

При выборе формата менеджер может: выбрать шаблон быстрого выбора, выбрать формат вручную из списка, скорректировать печатную область и технологические поля, сохранить настройку как шаблон.

ℹ️ Смежная тема (вне раскладки): у каждого принтера разная скорость печати в зависимости от плотности бумаги и формата. Эти данные нужны для производственного планирования (расчёт времени печати), а не для раскладки. Матрица скоростей печати (printer_speed_matrix) и коэффициенты форматов (format_speed_coefficients) описаны в их единственном владельце — ../06-production/equipment.md, раздел 2; алгоритм расчёта времени печати (класс PrintTimeCalculator) — в cutting-lamination-services.md, раздел 5. Здесь значения скоростей намеренно не дублируются.

Сводные правила (раздел 3.2 спецификации)#

  • Учёт обеих ориентаций макета.
  • Выбор максимального количества изделий на листе.
  • Учёт технологических полей принтера (непечатаемая область).
  • Учёт меток для плоттера: 8 мм (смещение от края) + 10 мм (безопасный отступ от меток до макета).
