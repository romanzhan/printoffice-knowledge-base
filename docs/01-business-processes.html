Бизнес-процессы (Event Map)

Полный путь клиента типографии PrintOffice — от первого контакта до закрытия заказа. Документ описывает этапы обработки заказа, различия B2C и B2B, правила автоматического выбора технологии печати, расчёт сроков и дедлайнов, а также процедуру закрытия заказа (УПД, отзывы).

1. Бизнес-процесс AS-IS (как есть сейчас)

Текущий процесс работы, описанный со слов владельца.

Источники привлечения клиента

  • Сайт компании.
  • Поисковые системы: Яндекс, Google.
  • Карты: Яндекс.Карты, 2ГИС, Google Maps.
  • Платная реклама: Яндекс.Директ.

После попадания на сайт клиент обычно переходит по ссылке в чат с менеджером в Telegram, где ведётся диалог: расчёты, уточнение сроков, ТЗ, стоимости.

Целевое изменение: перенести расчёт стандартных заказов на сайт, чтобы клиенты сами оформляли заказы, указывая все параметры. Менеджер подключается только когда ресурсов сайта недостаточно для полного расчёта или для ответов на вопросы клиента.

Текущий ход обработки заявки

После получения заявки в Telegram или MAX менеджер выясняет:

  • Есть ли готовый макет у клиента или его нужно подготовить.
  • Сроки выполнения заказа.
  • Тираж.
  • Используемые материалы, плотность бумаги (либо это баннер, печать на пластике).
  • Количество позиций в заказе.
  • Цветность: цветная или чёрно-белая печать.
  • Способ оплаты:
    • Физическое лицо — картой на сайте либо переводом на карту.
    • Юридическое лицо — по счёту (запрашивается ИНН для выставления счёта). Некоторые юрлица просят договор — он подписывается электронно по ЭДО Диадок.
  • Необходимость дополнительной обработки: скругление углов, ламинация (разной плотности, глянцевая/матовая и других видов).

Особый случай — бейджи: иногда нужно печатать одинаковые бейджи, а иногда все макеты разные — это сложнее и поэтому дороже.

Работа через подрядчиков

Собственной широкоформатной, офсетной и сувенирной печати у компании нет. Например, печать браслетов перезаказывается у подрядчика:

  1. Менеджер пишет подрядчику в Telegram, отправляет ТЗ.
  2. Подрядчик присылает стоимость и сроки.
  3. Компания добавляет свою наценку за вычетом всех расходов, включая доставку к себе в офис от подрядчика и налог 7%.
  4. Итоговая стоимость отправляется клиенту на согласование.
  5. После согласования берётся оплата, начинается выполнение заказа.
  • Широкоформатная печать: ТЗ отправляется партнёру, он делает просчёт, компания добавляет наценку. Цель — составить базу цен для широкоформата, чтобы не запрашивать цену каждый раз и сразу называть стоимость и сроки клиенту.
  • Офсетная печать: у партнёров есть сайт с выставленными ценами и сроками — цены просто списываются и добавляется наценка с учётом расходов на налоги и доставку. Таких партнёров три, у каждого свои цены и сроки.

Завершение работы по заказу

  1. Проверка качества продукции.
  2. Упаковка товара, чтобы не повредился.
  3. Нумерация количества позиций (чтобы при отгрузке ничего не перепутать).
  4. Согласование даты самовывоза и пункта выдачи (три офиса, можно забрать в любом) либо доставки: время, кто получает, до какого времени, телефон получателя.
  5. После выполнения — отправка закрывающих документов по ЭДО Диадок.
  6. Просьба оставить отзыв на Яндекс.Картах и в 2ГИС.

Требования к файлам (макетам)

  • Формат: PDF.
  • Вылеты (bleed) по 3 мм со всех сторон.
  • Разрешение: минимум 300 dpi.
  • Все слои сведены.
  • Шрифты переведены в кривые.

Макет проверяется. При необходимости правок:

  • Лёгкие правки — вносятся бесплатно.
  • Сложные правки — платно, либо клиент исправляет макет самостоятельно.

После подготовки макета делается принтскрин и согласуется с клиентом (проверка на ошибки). После согласования макета претензии по макету не принимаются.

Далее менеджер планирует производство: закладывает сроки, сверяет наличие материалов, загруженность производства.

Болевые точки (что нужно автоматизировать)

  • Автоматизация расчёта заказов через сайт — клиенты сами оформляют заказы, указывая все параметры.
  • Автоматическая выдача закрывающих документов.
  • Лёгкий поиск и повторение прошлых заказов.
  • Не терять макеты и заказы при повторном обращении.
  • Напоминания клиентам о компании, чтобы её не забывали.

2. Полный путь клиента (Customer Journey) — сводно

Этап Описание
1 Первый контакт (сайт, Telegram, email)
2 Выяснение параметров заказа
3 Работа с макетом (проверка, правки, согласование)
4 Выбор технологии печати (цифра / офсет / широкоформат)
5 Расчёт стоимости и сроков
6 Согласование и оплата
7 Производство
8 Доставка и выдача
9 Закрытие заказа

3. Детальный Event Map по этапам

Этап 1: Первый контакт

START: Клиент находит сайт или пишет в мессенджер (Telegram / WhatsApp)
  │
  ▼
EVENT: Клиент обращается в чат
  ├── IF через сайт   → открывается чат-виджет
  ├── ELSE IF Telegram → сообщение в Telegram-бота
  └── ELSE IF WhatsApp → сообщение в WhatsApp
  │
  ▼
EVENT: Менеджер отвечает (автоматическое приветствие)
  └── "Здравствуйте! Чем могу помочь? Расскажите, что вам нужно напечатать."
  │
  ▼
GOTO Этап 2

Этап 2: Выяснение параметров заказа

Параметры собираются последовательно: тип продукции → формат/размер → бумага/материал → цветность → тираж → наличие макета.

EVENT: Менеджер/бот выясняет параметры заказа
  ├── Формат? (A4, A5, A6, A3, SRA3, свой)
  │     └── IF свой → уточнить ширину и высоту в мм
  ├── Тираж?
  │     └── IF тираж < 10 → предупредить о минимальной стоимости
  ├── Цветность? (1+0, 1+1, 4+0, 4+4)
  └── Есть ли макет?
        ├── IF макета нет → Этап 3 (Дизайн)
        └── ELSE          → Этап 3 (Проверка макета)

2.1. Параметр: Тип продукции

Варианты: листовки, визитки, брошюры (на скобе / на пружине), наклейки (простые / стикерпаки / фигурные), плакаты, буклеты, конверты, календари, другое.

2.2. Параметр: Формат / Размер

Формат Размер (мм)
A4 210×297
A5 148×210
A6 105×148
A3 297×420
SRA3 320×450
Евро 98×210
Свой размер вводятся ширина и высота в мм

Проверка корректности формата:

  • Если формат больше максимального → "Слишком большой формат, нужна широкоформатная печать". Минимум для цифры — 20×20 мм; максимум — 320×450 мм.
  • Если формат меньше минимального → "Слишком маленький формат, нужна вырубка на плоттере".

2.3. Параметр: Бумага / Материал

По плотности Назначение
80 г/м² офсетная, для черновиков
130 г/м² мелованная, для листовок
200 г/м² плотная, для визиток
250 г/м² очень плотная, для обложек
300 г/м² дизайнерская
  • По типу поверхности: матовая, глянцевая, сатинированная.
  • По цвету: белая, цветная, дизайнерская (с фактурой).

Система проверяет наличие материала на складе; если нет — предупреждение и предложение альтернативы.

2.4. Параметр: Цветность

Код Значение
1+0 чёрно-белая односторонняя
1+1 чёрно-белая двусторонняя
4+0 цветная односторонняя
4+4 цветная двусторонняя

2.5. Параметр: Тираж

  • Если тираж < минимального (например, 10 шт; для брошюр — 20 шт) → предупреждение.
  • Если тираж > максимального для цифровой печати → рекомендация офсета (дешевле).
  • Если тираж промежуточный (нет в таблице цен) → "Цена будет рассчитана как за {ближайший_больший_тираж} шт".

Рекомендации по технологии в зависимости от тиража:

  • Тираж < 50 — оптимально цифровая печать.
  • Тираж > 5000 — рекомендуется офсетная печать (дешевле).

2.6. Параметр: Наличие макета

  • "Да, есть" → переход к проверке макета.
  • "Нет, нужен дизайн" → переход к этапу дизайна.

Валидация параметров

  • Размер < 20×20 мм → вырубка на плоттере.
  • Размер > 320×450 мм → широкоформатная печать.
  • Тираж < 10 шт → ниже минимального; для брошюр < 20 шт.
  • Бумага плотностью < 80 г/м² с цветностью 4+4 → рекомендовать бумагу от 130 г/м².

Таблица параметров для CRM

CREATE TABLE order_parameters (
    id INT PRIMARY KEY AUTO_INCREMENT,
    order_id INT NOT NULL,
    -- Тип продукции
    product_type VARCHAR(50) NOT NULL,   -- 'flyer', 'business_card', 'brochure', 'sticker'
    -- Формат
    format VARCHAR(20),                  -- 'A4', 'A5', 'custom'
    custom_width INT,
    custom_height INT,
    -- Бумага
    paper_id INT,
    paper_density INT,
    paper_type VARCHAR(50),              -- 'matte', 'glossy', 'offset'
    paper_color VARCHAR(50),
    -- Цветность
    color_type VARCHAR(10),              -- '1+0', '1+1', '4+0', '4+4'
    -- Тираж
    quantity INT NOT NULL,
    -- Макет
    has_maket BOOLEAN DEFAULT false,
    maket_ready BOOLEAN DEFAULT false,
    -- Дополнительно
    binding_type VARCHAR(20),            -- для брошюр
    cutting_type VARCHAR(50),            -- для наклеек
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (order_id) REFERENCES orders(id),
    FOREIGN KEY (paper_id) REFERENCES materials(id)
);

Этап 3: Работа с макетом

3.1. Дизайн (макета нет)

Выясняются детали для дизайна:

  • Фирменные цвета → запросить коды (Pantone, CMYK).
  • Логотип:
    • В векторе (AI, EPS, SVG) → принимаем.
    • В TIF/PNG → предупредить о возможной потере качества.
    • Нет → запросить у клиента.
  • Пожелания по дизайну → записать видение, стиль, примеры.
  • Отправить бриф клиенту (ссылка на Google Form или PDF-опросник).
  • Попросить референсы.

Оценка стоимости и сроков дизайна:

Тип Срок Коэффициент цены
Срочно 1 день ×1.5
Обычный (дизайнер не загружен) 2–3 дня ×1.0
Сложный макет 3–7 дней ×1.3
Дизайн не наш профиль — ОТКАЗ

После согласования стоимости и сроков выставляется счёт на оплату дизайна:

  • Юрлицо → счёт на email.
  • Физлицо → ссылка на оплату или QR-код.

Ожидается оплата; при отсутствии — напоминание через 24 часа. После оплаты — приступаем к дизайну, затем переход к согласованию макета.

3.2. Проверка макета (есть у клиента)

Проверка формата файла:

  • PDF → OK.
  • CDR (CorelDRAW) → сделать скриншот для проверки.
  • Иначе → перевести в PDF или запросить PDF.

Чек-лист соответствия техническим требованиям:

  • Размеры макета соответствуют заказанному формату.
  • Вылеты (bleed) = 3 мм со всех сторон.
  • Разрешение = 300 dpi.
  • Цветовая модель = CMYK.
  • Шрифты в кривых или приложены.
  • Слои не сведены (если нужны правки).
  • Изображения не пикселизируются.

Маршрутизация по результату:

  • Все OK → согласование.
  • Простые правки (цвет, размер, вылеты) → правим сами → согласование.
  • Сложные правки (нет шрифтов, слои сведены, JPEG) → отправляем на доработку клиенту (техтребования, шаблон PDF с метками), ждём исправленный макет → повторная проверка.

3.3. Согласование макета

Макет отправляется клиенту на согласование с уведомлением: "Пожалуйста, проверьте макет перед печатью. После утверждения внести правки будет невозможно."

  • Клиент утвердил → Этап 4.
  • Нужны правки → возврат к правкам.
  • Иначе → заказ приостановлен.

Этап 4: Выбор технологии печати

EVENT: Определяем способ печати
  ├── IF тираж < 500 ИЛИ формат ≤ SRA3 → цифровая (печатаем сами)
  ├── ELSE IF тираж > 500 И формат ≤ SRA3 → офсет (партнёр ГМ Принт)
  ├── ELSE IF формат > SRA3 → широкоформат (партнёр)
  └── ELSE → ручной просчёт (отправляем партнёру)

Подробные правила — см. раздел 5.

Этап 5: Расчёт стоимости и сроков

5.1. Цифровая печать (своя)

sheets = ceil(тираж / items_per_sheet)
cost   = sheets × (print_price + paper_price)
margin = find_margin_by_sheets(sheets)
price  = cost × (1 + margin/100)

Добавление доставки:

  • Самовывоз → 0 руб.
  • Курьер → по весу и адресу.
  • Доставка партнёром → цена партнёра.

5.2. Офсетная печать (партнёр ГМ Принт)

Получение цены от партнёра:

  • Цена есть в нашей таблице → берём оттуда.
  • Тираж большой → отправляем партнёру на ручной просчёт.
  • Иначе → интерполяция по ближайшим тиражам.

Формирование итоговой цены:

partner_price = цена от партнёра
delivery      = стоимость доставки (бесплатно до 100 кг в МКАД)
tax           = partner_price × 7%
our_markup    = partner_price × markup_percent
price         = partner_price + delivery + tax + our_markup

Этап 6: Согласование и оплата

Отправка клиенту итоговой стоимости и сроков, затем согласование способа получения:

  • Самовывоз: уточнить офис (адрес, график работы).
  • Доставка: запросить адрес, время (с и до), телефон получателя, рассчитать стоимость доставки.

Выбор способа оплаты:

  • Юрлицо: запросить ИНН → выставить счёт на email → при необходимости договор по ЭДО → ждать оплату.
  • Физлицо: отправить номер заказа → ссылку на сайт с быстрой оплатой (карта) или QR-код.

Проверка статуса оплаты:

  • Оплачено → Этап 7.
  • Не оплачено и клиент новый → заказ НЕ запускается.
  • Не оплачено и клиент постоянный → МОЖЕМ запустить + задача "Напомнить об оплате через 3 дня".

Этап 7: Производство

  1. Подготовка макета к печати: конвертация в PDF (если не PDF), проверка CMYK, добавление меток (для контурной резки), раскладка на листе.
  2. Печать. При проблеме по вине макета — остановить печать и отправить на исправление клиенту либо править самим.
  3. Контроль качества печати: цветопередача, чёткость, отсутствие дефектов; при браке — перепечатка.
  4. Послепечатная обработка: резка (гильотинная/плоттерная), ламинация, биговка/фальцовка, сборка (если брошюра), упаковка.
  5. Финальная проверка качества: сверка с макетом, проверка количества, фото- и видеоотчёт.
  6. Маркировка заказа: ФИО клиента / название организации, количество позиций, номер позиции (если сборный заказ), номер заказа.

Этап 8: Доставка и выдача

Уведомление клиента о готовности, затем выбор способа доставки:

  • Самовывоз: клиент приезжает в офис (сообщаются адрес, график, телефон) → выдача.
  • Доставка курьером (Яндекс, Достависта): выбор типа курьера (пеший до 5 кг / на авто от 5 кг) → созвон с клиентом для подтверждения времени → передача заказа курьеру.

Особые случаи:

  • Сборный заказ (часть мы, часть партнёр): собираем в одной точке (наш офис), проверяем качество и комплектацию, маркируем, отгружаем клиенту.
  • Доставка от партнёра ГМ Принт: бесплатно в пределах МКАД. Широкоформатную печать забираем сами или вызываем курьера.

Перед отгрузкой проверяем количество мест и комплектацию заказа.

Этап 9: Закрытие заказа

  1. Если была оплата по счёту: создаём УПД (универсальный передаточный документ), отправляем по ЭДО. По запросу клиента отправляем УПД (часто получение не отслеживается).
  2. Просим отзыв: ссылки на Яндекс.Карты, 2ГИС, Google Maps.
  3. Благодарим за заказ: "Спасибо за заказ! Будем рады видеть вас снова!"
  4. Закрываем заказ в CRM: статус "Выполнен", дата закрытия.

Различие по типу клиента при закрытии:

  • B2C: чек на email/телефон.
  • B2B: УПД, акт, закрытие.

⚠️ ОТКРЫТЫЙ ВОПРОС: Расхождение по системе ЭДО для закрывающих документов. В описании AS-IS и в Event Map указан Диадок («Диадок»), а в визуальной схеме Этапа 9 — Эльба. Нужно зафиксировать единого провайдера ЭДО для УПД.

4. Различия B2C и B2B + быстрые заказы

4.1. Классификация клиентов

Характеристика B2C (физические лица) B2B (юридические лица)
Канал Чаще офлайн в офисе Чаще онлайн (сайт, почта, мессенджеры)

Жизненный цикл Минуты → часы → редко дни Дни → недели
Оплата На месте (карта/наличные) или быстрая ссылка По счёту (безналичный расчёт)
Документооборот Минимальный (чек, без договора) Договор, УПД, счёт, акт
Доставка Чаще самовывоз (тут же в офисе) Курьер, транспортная компания

4.2. Типы заказов по срочности и сложности

Тип Примеры Срок Клиент Оплата
Срочные / моментальные (B2C) Копирование, распечатка, печать 1–10 листов, срочная визитка 1 мин – 2 ч В офисе, ждёт на месте Сразу на месте
Обычные (B2C и B2B) Листовки, визитки, наклейки (не срочно) 1–3 дня В офисе или онлайн Онлайн или по счёту
Сложные / крупные (B2B) Брошюры, каталоги, крупные тиражи, дизайн + печать 3–14 дней Онлайн, удалённо По счёту, договор, УПД

Для срочных B2C задача — минимум действий в CRM, быстрая обработка.

4.3. Концепция «Быстрый заказ» (B2C)

Минимальный набор полей: тип fast, клиент (имя + телефон, можно без регистрации), список услуг, итоговая сумма, способ оплаты (cash/card/online), статус сразу completed после оплаты.

const FAST_ORDER = {
    type: 'fast',                    // fast / regular / complex
    client: {
        name: 'Иван Петров',         // только имя, можно без регистрации
        phone: '+7 999 123 45 67',
        isNew: true
    },
    services: [
        { name: 'Копирование A4', quantity: 20, price: 10 },
        { name: 'Распечатка',     quantity: 5,  price: 15 }
    ],
    total: 275,
    payment: 'cash',                 // cash / card / online
    status: 'completed',             // сразу completed после оплаты
    createdAt: new Date(),
    completedAt: new Date()
};

Процесс быстрого заказа в офисе: клиент называет услугу → менеджер (или клиент через терминал) выбирает услугу в упрощённом интерфейсе → клиент указывает количество → система мгновенно показывает сумму → оплата (карта/наличные) → выполнение → выдача → система создаёт «быстрый заказ» со статусом completed (время создания = время завершения) → опционально чек на email или телефон.

Прайс быстрого выбора услуг (B2C):

Услуга Цена
Копировать A4 5 руб/лист
Копировать A3 10 руб/лист
Распечатать A4 15 руб/лист
Распечатать A3 25 руб/лист
Фото 50 руб/шт

Канал Назначение Интеграция

Телефон B2C: консультации, уточнение адреса/времени. B2B: обсуждение крупных заказов Запись звонков, привязка к заказу

4.5. Статусы заказов и автоматическое определение типа заказа

Полные статусные словари B2C_STATUSES / B2B_STATUSES и класс автоопределения OrderTypeDetector (с признаками isFastB2C / isComplexB2B, порогами quantity < 100 / quantity > 500) вынесены в ядро CRM — см. CRM: обзор → Типы заказов и их автоопределение.

Кратко для бизнес-процесса:

  • B2C (быстрые заказы): pending → processing → ready → completed (плюс cancelled). Первый статус — pending («клиент ещё не оплатил»), как в словаре B2C_STATUSES первоисточника (raw/architecture.md:964); new относится к B2B-цепочке.
  • B2B (сложные заказы): new → calculation → proposal_sent → proposal_accepted → payment_pending → paid → production → ready → delivery → completed (плюс cancelled).
  • Тип заказа (fast / complex / regular) определяется на входе автоматически по типу клиента, услугам, тиражу, потребности в дизайне, срочности, способу оплаты и способу получения; от типа зависят цепочка статусов, необходимость договора/счёта и авто-завершение.

⚠️ ОТКРЫТЫЙ ВОПРОС: Несогласованность порога тиража для определения «быстрого» заказа. В детекторе типа заказа граница quantity < 100, тогда как граница выбора цифровой печати — тираж < 500 (раздел 5), а рекомендация офсета начинается при тираже > 5000. Нужно согласовать пороги.

4.6. Итоговая схема обработки по типам

ВХОДНОЙ ЗАКАЗ → ОПРЕДЕЛЕНИЕ ТИПА КЛИЕНТА И ЗАКАЗА
   ├── B2C (быстрый): копирование, распечатка, фото, срочная полиграфия
   │     → упрощённый процесс: минимум полей, оплата на месте, выдача сразу, статус completed
   ├── B2C (обычный): листовки (малый тираж), визитки, наклейки
   │     → полный процесс: все поля, оплата онлайн/счёт, доставка/самовывоз, полная цепочка статусов
   └── B2B (сложный): крупные тиражи, брошюры, дизайн, каталоги
         → расширенный процесс: все поля + договор, счёт, УПД, ЭДО, доставка, расширенные статусы
ЗАКРЫТИЕ: B2C — чек на email/телефон; B2B — УПД, акт, закрытие

5. Правила автоматического выбора технологии печати

5.1. Концепция

На вход подаются параметры заказа (формат, тираж, бумага, цветность, срочность, доп. услуги). Система автоматически рассчитывает варианты по трём технологиям (цифра / офсет ГМ Принт / широкоформат), сравнивает по цене, срокам, качеству, отмечает рекомендуемый вариант, а клиент (или менеджер) выбирает.

5.2. Условия выбора технологий

Макс. высота формата, мм 450 450 5000 (мин. 451)
Мин. тираж 1 500 1

Цветность 1+0, 1+1, 4+0, 4+4 1+0, 1+1, 4+0, 4+4 4+0, 4+4
Срочность до 1 дня минимум 3 дня —

5.3. Алгоритм выбора

  1. Проверить условия каждой технологии (checkConditions).
  2. Для подходящих рассчитать цену и сроки.
  3. Отсортировать варианты по цене (по возрастанию).
  4. Отметить рекомендуемый (по умолчанию — самый дешёвый). Эвристика isRecommended: цифровая, если цена < 5000 руб; офсет, если цена < 10 000 руб.
  5. Вернуть: available (все варианты), recommended, alternatives.

6. Сроки выполнения и дедлайны

6.1. Единый конструктор сроков

Гибкий конструктор настраивается для каждого типа печати (офсет, цифра, широкоформат) и для каждого партнёра отдельно. Хранится в одной таблице с полем rule_type. Базовые настройки: базовый срок (дни), время приёма заказов, рабочие дни, условия увеличения (тираж, услуги), доставка по городам, возможность субботней доставки.

6.2. Базовые правила (офсет, ГМ Принт)

  • Время приёма заказов: до 16:00 — заказы после этого времени обрабатываются на следующий день.
  • Базовый срок готовности: 1 день.
  • Рабочие дни: Пн–Пт (Сб и Вс — нерабочие для производства/готовности).

6.3. Дополнительные условия (увеличение срока)

  • Если тираж > 5000 шт → +1 день.
  • Если есть любые дополнительные услуги → +1 день.
  • Если услуга «Ламинация» → +1 день.
  • Если услуга «Биговка» (scoring) → +0.5 дня.

6.4. Доставка по городам и субботняя доставка

Зона Доп. срок Суббота
Москва (МКАД) +0 дней не возможна
Московская область +2 дня возможна (+500 руб)
Другие города РФ +3 дня не возможна

Субботняя доставка не добавляет дней к сроку, только доплату (+500 руб).

6.5. Примеры расчёта (офсет)

Заказ Готовность
Тираж 3 000, без услуг, Москва 1 день
Тираж 6 000, без услуг, Москва 2 дня
Тираж 3 000, ламинация, Москва 2 дня
Тираж 3 000, без услуг, МО 3 дня
Тираж 6 000, ламинация, доставка в МО 5 дней (базовый 1 + тираж 1 + ламинация 1 + МО 2)

6.6. Настройка для цифровой печати (отличия)

  • Базовый срок: 1 день.
  • Время приёма: до 18:00.
  • Зависимости от тиража нет (тираж > 1000 → +0 дней).
  • Ламинация: +0.5 дня (на цифре быстрее).
  • Доставка: как у офсета, можно настроить отдельно.

⚠️ ОТКРЫТЫЙ ВОПРОС: Разное время приёма заказов для разных технологий: офсет — до 16:00, цифра — до 18:00. Нужно подтвердить корректность и единый ли cutoff в публичных коммуникациях с клиентом.

6.7. SLA партнёра ГМ Принт

  • Готовность — на следующий день (не «через день»), кроме воскресенья.
  • Тираж до 5000 — готовность на следующий день.
  • Постобработка — +1 день.
  • Доставка от службы ГМ Принт не работает по субботам и воскресеньям; в субботу заказ от них можно доставить своим курьером.
  • Самовывоз работает в субботу и воскресенье.
  • Доставка > 100 кг — платная.
  • Доставка в МО — платная.

6.8. Структура БД и расчёт сроков

CREATE TABLE deadline_rules (
    id INT PRIMARY KEY AUTO_INCREMENT,
    rule_name VARCHAR(255) NOT NULL,
    rule_type ENUM('offset','digital','wide') NOT NULL,   -- тип печати

    base_days INT DEFAULT 1,                               -- базовый срок

    conditions JSON,                                       -- условия увеличения срока
    delivery_rules JSON,                                   -- настройки доставки
    is_active BOOLEAN DEFAULT true,

    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

        "Москва": {"days": 0, "saturday_allowed": true, "saturday_price": 500},

Алгоритм расчёта (DeadlineCalculator):

  1. Берётся базовый срок (base_days).
  2. Если время заказа позже cutoff_time → +1 день.
  3. Применяются условия из правил (тираж, услуги) → суммируются add_days.
  4. Прибавляются дни доставки по городу.
  5. Субботняя доставка (если запрошена и разрешена) — добавляет доплату saturday_price, но НЕ добавляет дней.
  6. Итоговая дата считается по рабочим дням (working_days), нерабочие дни пропускаются.

В CRM срок отображается с разбивкой по составляющим и возможностью ручной корректировки (с указанием причины, например «Клиент просил отсрочку»).

⚠️ ОТКРЫТЫЙ ВОПРОС: По МКАД (Москва) субботняя доставка в SLA ГМ Принт указана как «доставка не работает по Сб/Вс», но в delivery_rules для Москвы saturday_allowed: true (+500 руб). Нужно уточнить, разрешена ли субботняя доставка по Москве и каким именно перевозчиком (служба ГМ Принт vs собственный курьер).
