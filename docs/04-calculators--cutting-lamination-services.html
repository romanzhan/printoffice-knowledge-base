Резка, ламинация и доп. услуги

Спецификация расчёта постпечатных операций типографии: гильотинная резка (Boway BW-4908 V9), плоттерная контурная резка (Mimaki CG-130), пакетная и рулонная ламинация, а также универсальная модель ценообразования дополнительных услуг по формуле (материал + время × ставка) × (1 + маржа).

1. Два типа резки в производстве

В производстве используются два принципиально разных способа резки. Выбор зависит от типа изделия.

Тип резки Оборудование Для каких изделий Принцип
Гильотинная Резак (гильотина) Boway BW-4908 V9 Листовки, визитки, прямоугольные изделия Режет стопу листов за один проход
Плоттерная Плоттер (Mimaki CG-130, Roland и др.) Наклейки, фигурная резка, контур Режет каждый лист отдельно по контуру

Ключевые различия:

  • Гильотина режет всю стопу одним резом → время зависит от числа резов и числа стоп, а не от тиража поштучно.
  • Плоттер режет каждый лист отдельно → время линейно растёт с тиражом и сильно зависит от сложности контура.

⚠️ ОТКРЫТЫЙ ВОПРОС: листовки в стопе режутся на гильотине, а не на плоттере — это явная поправка к более ранним черновикам, где резка наклеек/листовок описывалась через плоттер. В финальной модели плоттер используется только для фигурной/контурной резки наклеек.

2. Гильотинная резка (Boway BW-4908 V9)

2.1. Параметры резака

const GUILLOTINE_SPECS = {
    model: "Boway BW-4908 V9",
    // Максимальные параметры
    maxCutWidth: 490,      // мм — ширина реза
    maxCutDepth: 450,      // мм — глубина реза (размер бумаги)
    maxStackHeight: 80,    // мм — максимальная высота стопы
    // Технические характеристики
    clampPressure: 'adjustable',  // давление прижима
    cutSpeed: 0.5,                // минут на один рез (среднее)
    hasProgrammableCutter: true,  // программируемые резы
    accuracy: 0.1                 // мм точность
};

2.2. Толщина бумаги по плотности

Высота стопы вычисляется из количества листов и толщины одного листа. Толщина определяется по плотности.

Полная таблица толщины по плотности (80=0.10 … 400=0.48), формула density × 0.0011, значение по умолчанию 0.15 мм и сведённое расхождение между таблицей и формулой — в каноническом справочнике: Оборудование и нормативы, раздел 8.1 «Толщина и вес бумаги». При расчёте высоты стопы используется табличное значение толщины (например, 130 г/м² → 0.16 мм; 80 г/м² → 0.10 мм).

2.3. Высота стопы и коэффициент уплотнения

Реальная высота стопы меньше теоретической из-за уплотнения (бумага сжимается под прижимом).

thicknessPerSheet = paperThickness[density]   // мм/лист
theoreticalHeight = sheets × thicknessPerSheet
actualHeight      = theoreticalHeight × compressionFactor
compressionFactor = 0.85   // реальная высота на 15% меньше
fitsInGuillotine  = actualHeight <= 80 мм

Максимальное количество листов в стопе для заданной плотности:

maxSheetsPerStack = floor( maxStackHeight / (thickness × compressionFactor) )
                  = floor( 80 / (thickness × 0.85) )

2.4. Таблица высоты стопы по плотности

Плотность (г/м²) Толщина (мм) Макс. листов в стопе Высота стопы (мм) Коэф. времени
80 0.10 80 / (0.10×0.85) = 941 80 1.5
130 0.16 80 / (0.16×0.85) = 588 80 1.5
200 0.24 80 / (0.24×0.85) = 392 80 1.5
250 0.30 80 / (0.30×0.85) = 313 80 1.5
300 0.36 80 / (0.36×0.85) = 261 80 1.5
350 0.42 80 / (0.42×0.85) = 224 80 1.5

2.5. Количество резов для раскладки

Для сетки cols × rows число резов рассчитывается с оптимальной стратегией — сначала режем по длинной стороне (по полосам), затем каждую полосу на части.

cutsForGrid(cols, rows):
    if cols >= rows:
        return (cols - 1) + (rows - 1) × cols   // сначала вертикальные резы
    else:
        return (rows - 1) + (cols - 1) × rows   // сначала горизонтальные резы

Примеры:

  • сетка 4×2 → (4−1) + (2−1)×4 = 3 + 4 = 7 резов;
  • сетка 2×4 → (2−1) + (4−1)×2 = 1 + 6 = 7 резов.

При гильотинной резке режется сразу вся стопа, поэтому число резов не умножается на число листов в стопе.

Альтернативный метод (резы только в одном направлении, если изделия уже нарезаны на полосы):

  • горизонтальные: rows − 1;
  • вертикальные: cols − 1.

2.6. Коэффициент высоты стопы

Чем выше стопа, тем медленнее рез (выше давление, аккуратнее проход). Эмпирическая формула:

stackHeightFactor = 1.0 + (actualHeight / maxStackHeight) × 0.5
                  = 1.0 + (actualHeight / 80) × 0.5

При высоте 80 мм коэффициент = 1.5; при высоте ~20 мм ≈ 1.125.

2.7. Итоговая формула времени

const guillotineTime = (sheets, density, cols, rows, timePerCut = 0.5, setupTime = 3) => {
    const maxSheets = maxSheetsInStack(density);                 // floor(80 / (thickness×0.85))
    const stacks    = Math.ceil(sheets / maxSheets);
    const actualSheetsPerStack = Math.ceil(sheets / stacks);
    const thickness = paperThickness(density);
    const stackHeight = actualSheetsPerStack × thickness × 0.85;
    const heightFactor = 1.0 + (stackHeight / 80) × 0.5;
    const cuts = cutsForGrid(cols, rows);
    return stacks × cuts × timePerCut × heightFactor + setupTime;
};

Параметры по умолчанию:

  • timePerCut = 0.5 мин на один рез;
  • setupTime = 3 мин на настройку (приладку) на каждый заказ.

2.8. Примеры расчёта

Пример 1. Листовки А5, тираж 1000 шт, бумага 130 г/м², SRA3 320×450, раскладка 4×2.

1. Толщина 130 г/м² = 0.16 мм
2. Макс. листов в стопе: 80 / (0.16 × 0.85) = 588
3. Листов 125 (1000/8) → 1 стопа (помещается)
4. Резов на лист: 4×2 → (4-1)+(2-1)×4 = 7
5. Коэф. высоты: 1 + (20/80)×0.5 = 1.125
6. Время: 1 × 7 × 0.5 × 1.125 + 3 = 3.94 + 3 = 6.94 мин

Пример 2. Тираж на 2000 листов, плотная бумага 300 г/м², раскладка 2×4.

1. Толщина 300 г/м² = 0.36 мм
2. Макс. листов в стопе: 80 / (0.36 × 0.85) = 261
3. 2000 листов → 8 стоп (2000/261 = 7.66 → 8)
4. Резов на лист: 2×4 → (2-1)+(4-1)×2 = 7
5. Высота стопы ~250 × 0.36 × 0.85 = 76.5 мм → коэф. 1.48
6. Время: 8 × 7 × 0.5 × 1.48 + 3 = 41.44 + 3 = 44.44 мин

Пример 3 (упрощённая формула, листовки А5, тираж 1000). На SRA3 8 шт листовок А5 → 125 листов; нужно 4 реза (3 горизонтальных + 1 вертикальный); по 100 листов в стопе:

guillotineTime(125, 4, 100):
  стоп = ceil(125/100) = 2
  2 × 4 × 0.5 + 3 = 4 + 3 = 7 мин

2.9. Оптимизация размера стопы

Алгоритм optimizeStacks перебирает размеры стопы (от 50 листов до максимума с шагом 10) и выбирает вариант с минимальным суммарным временем резки. Менеджер может задать число листов в стопе вручную (sheetsPerStack), тогда авторасчёт игнорируется.

3. Плоттерная контурная резка (Mimaki CG-130)

3.1. Параметры плоттера

Параметр Значение Влияние
Скорость перемещения (макс) 800 мм/сек Максимальная скорость на прямых участках
Скорость резки 200–500 мм/сек Зависит от материала и сложности
Ускорение/торможение 0.5–1 сек На каждом углу/повороте
Переходы между элементами 1–3 сек Перемещение ножа между контурами

3.2. Скорость резки по сложности

const PLOTTER_SPEED_SETTINGS = {
    machineMaxSpeed: 800, // мм/сек — макс. скорость перемещения
    cuttingSpeed: {
        simple:      { speed: 500, cornerPenalty: 1.2, description: 'Простые наклейки, этикетки' },
        medium:      { speed: 350, cornerPenalty: 1.5, description: 'Овалы, круги, скругленные углы' },
        complex:     { speed: 250, cornerPenalty: 2.0, description: 'Фигурные наклейки, логотипы' },
        veryComplex: { speed: 150, cornerPenalty: 3.0, description: 'Стикерпаки, мелкие элементы' }
    },
    timePenalties: {
        acceleration:    0.5,  // сек на разгон/торможение
        cornerSlowdown:  0.3,  // сек на каждый угол
        moveBetweenItems:1.0,  // сек на переход между элементами
        pierceStart:     2.0   // сек на прокол материала в начале (один раз на лист)
    }
};

Сложные элементы режутся на пониженной скорости (например, 500 мм/сек — простые, 250 мм/сек — сложные); скорость можно задать и вручную.

3.3. Расчёт периметра элемента

prim = (shape == 'circle') ? π × diameter : 2 × (width + height)

3.4. Формула времени резки одного листа

Время на лист (в секундах) складывается из резки контуров, штрафа на углы, переходов между элементами и стартового прокола:

totalPathLength = Σ периметров всех элементов на листе
cuttingTime  = totalPathLength / speed                      // резка
cornerTime   = cornerCount × 0.3                            // углы (0.3 сек/угол)
moveTime     = (itemsPerSheet − 1) × 1.0                    // переходы
startTime    = 2.0                                          // стартовый прокол (1 раз на лист)
totalSheetTime = cuttingTime + cornerTime + moveTime + startTime   // сек

3.5. Формула времени на весь тираж

Каждый лист режется отдельно (не стопой):

const plotterTime = (itemsPerSheet, sheets, complexity) => {
    const speed = { simple: 500, medium: 350, complex: 250, veryComplex: 150 }[complexity];
    const perimeterPerItem = 2 * (itemWidth + itemHeight); // или π·d для круга
    const perSheet =
        (itemsPerSheet * perimeterPerItem) / speed +  // резка
        (itemsPerSheet * 4 * 0.3) +                   // углы (по умолчанию 4 на элемент)
        (itemsPerSheet - 1) * 1.0 +                   // переходы
        2;                                            // старт
    return (perSheet * sheets) / 60 + 5;              // минуты, +5 мин настройка
};

Время настройки плоттера: setupTime = 5 мин.

3.6. Примеры расчёта

Пример 1. Простая наклейка А5 (210×148 мм), 1 шт на листе, 100 листов, simple.

Периметр: 2×(210+148) = 716 мм
Скорость: 500 мм/сек → 716/500 = 1.43 сек
Углы: 4×0.3 = 1.2 сек; Переходы: 0; Старт: 2 сек
Итого на лист: 4.63 сек
На 100 листов: 463 сек = 7.7 мин + настройка 5 мин = 12.7 мин

Пример 2. Стикерпак А5: 20 наклеек 30×30 мм на листе, 50 листов, veryComplex.

Периметр одного: 2×(30+30) = 120 мм; общий путь: 20×120 = 2400 мм
Скорость: 150 мм/сек → 2400/150 = 16 сек
Углы: 20×4×0.3 = 24 сек (80 углов); Переходы: 19×1.0 = 19 сек; Старт: 2 сек
Итого на лист: 61 сек
На 50 листов: 3050 сек = 50.8 мин + настройка 5 мин = 55.8 мин

Пример 3. Наклейки А5 фигурные, 1000 шт по 1 на листе, complex (250 мм/сек).

Периметр 1 шт: 716 мм; общий путь: 716 × 1000 = 716 000 мм
Время резки: 716000/250 = 2864 сек
Углы: 4000 × 0.3 = 1200 сек; Переходы: 999 × 1.0 = 999 сек; Старт: 2 × 1000 = 2000 сек
ИТОГО: 7063 сек = 117.7 мин + настройка 5 мин = 122.7 мин

3.7. Альтернативная модель: резка стопами (упрощённая)

Для простых прямоугольных изделий, режущихся фактически стопами на гильотине, применяется укрупнённый норматив «время на стопу»:

stacks = ceil(quantity / sheetsPerStack)   // sheetsPerStack по умолчанию 50
time   = timePerStack × stacks
Тип изделия Сложность Время на стопу
Простая (прямоугольник) simple 2 мин
Средняя medium 3 мин
Сложная complex 5 мин
Очень сложная (стикерпак) very_complex 8 мин

Таблица сравнения (тираж 1000 шт, по 50 листов в стопе → 20 стоп):

Тип изделия Формат Время/стопу Стоп Общее время
Наклейка А5 простая А5 2 мин 20 40 мин
Наклейка А5 фигурная А5 3 мин 20 60 мин
Стикерпак А5 А5 8 мин 20 160 мин
Наклейка круглая d=50 — 2.5 мин 20 50 мин

⚠️ ОТКРЫТЫЙ ВОПРОС: в исходниках сосуществуют две несогласованные модели плоттерной резки — детальная (путь/скорость + углы + переходы + старт, посекундно) и укрупнённая (фиксированное «время на стопу»). Нужно решить, какая является основной, либо как они комбинируются (детальная — для плоттера, укрупнённая — для стоп на гильотине). Также детальная формула per-item (perimeter/baseSpeed × detail_penalty × cornerFactor) давала нереалистично большое время на штуку (2.39 мин на наклейку А5), что было признано ошибкой — реально режется стопа.

Сложность формы Коэффициент 1–3 Прямоугольник (1) vs сложная форма (2.5)

Справочник типов сложности (поля: базовая скорость мм/мин, штраф за углы, штраф за детали):

Тип Код Скорость (мм/мин) corner_penalty detail_penalty
Простая (прямоугольник) simple 300 1.0 1.0
Средняя (овалы, круги) medium 250 1.2 1.1
Сложная (фигурные) complex 200 1.5 1.3

⚠️ ОТКРЫТЫЙ ВОПРОС: единицы скорости резки расходятся между фрагментами — в справочнике сложности скорость указана в мм/мин (300/250/200/150), а в параметрах плоттера Mimaki — в мм/сек (500/350/250/150). Требуется унификация единиц и значений.

Для стикерпаков добавляется время на внутренние элементы: + innerElementsCount × 0.1 мин (либо × time_per_inner_element).

4. Раскладка макетов и вылеты для плоттерной резки

4.1. Правила вылетов

  • Стандартный вылет — 2 мм с каждой стороны (может регулироваться менеджером); встречаются также 1 мм или 3 мм.
  • Для круглого элемента вылет прибавляется к диаметру: width = diameter + bleed × 2.
  • Контур реза может идти «в ноль» к размеру макета — тогда вылеты не требуются.
  • Для прямоугольника: width = item.width + bleed × 2, height = item.height + bleed × 2.

4.2. Четыре сценария плоттерной резки

Сценарий Вылеты Расстояние между элементами Тип реза Применение
А 2 мм вплотную (0 мм) контурный Стандартный случай, каждый элемент по контуру
Б 0 мм вплотную (0 мм) прямой (линиями) Быстрая резка, требуется однородный/белый фон
В 2 мм мин. 2 мм контурный Сложные макеты, нужен зазор
Г переменный по ситуации любой Ручной режим менеджера

Предустановленные настройки:

Название Вылет Зазор Тип реза Фон Приоритет
Стандартная контурная 2 0 contour any 10
Быстрая линейная 0 0 straight_lines white 20
С зазорами 2 2 contour any 30
Экономная 1 0 contour any 40

Логика выбора:

  • Контурная — вырезается каждый элемент; при сложных формах рекомендуется зазор 0–2 мм между элементами для движения ножа.
  • Линейная (прямой рез) — элементы ставятся вплотную, режутся сквозными линиями; быстрее, но фон должен быть белым или однородной заливкой (макет без фона). Типовой приём: клиент присылает макет 40×40 мм, добавляются вылеты по 2 мм → 44×44 мм, элементы располагаются вплотную и режутся «в −2 мм» по контуру; реже режут 40×40 мм поперечными линиями (быстрее, но требуется однородный фон).
  • При обычной поэлементной резке расстояние между элементами должно быть не ближе 2 мм.

4.3. Область раскладки на листе

Площадь раскладки с приводочными метками плоттера (markerOffset = 8 мм + safetyGap = 10 мм → для SRA3 область макетов 284 × 414 мм, начало координат (18, 18)), формула и функция calculateLayoutArea — см. Раскладка на листе и вылеты, раздел «Метки плоттера для контурной резки» (этот файл — владелец расчёта раскладки, меток и вылетов).

Раскладка в полученной области:

  • контурная: cols = floor(area.width / (itemW + gap)), rows = floor(area.height / (itemH + gap));
  • линейная: вплотную, дополнительно считается план реза: horizontalCuts = rows − 1, verticalCuts = cols − 1.

4.4. Сравнение эффективности раскладки (элемент 40×40, лист 320×450, область макетов 284×414, см. раздел 4.3)

Сценарий Размер с вылетами Зазор На листе Время резки Подходит для
А (контур) 44×44 0 6×9 = 54 шт высокое Любые макеты
Б (линейный) 40×40 0 7×10 = 70 шт низкое Белый/однородный фон
В (с зазором) 46×46 (44+2) 2 мм 6×9 = 54 шт высокое Сложные формы

Экономия в сценарии Б: на 1000 шт нужно 1000/70 = 15 листов против 1000/54 = 19 листов (сценарий А) → экономия 4 листа (≈21%) и более быстрая резка.

Альтернативные форматы бумаги для того же элемента 40×40 (из UI-мокапа менеджера, сценарий А, контур 44×44):

Формат Размер листа (мм) Раскладка На листе

Менеджер выбирает формат в интерфейсе; для большого тиража банерный формат резко сокращает число листов.

4.5. Ручная правка количества раскладки и перерасчёт при смене параметров

Менеджер может выбрать другой принтер, формат бумаги (по умолчанию SRA3), отредактировать количество элементов в раскладке и техническую часть задания. При изменении любого параметра выполняется полный перерасчёт раскладки, технологических процессов и стоимости.

Зачем нужна ручная правка количества раскладки — конфликт «печатный лист → ламинация». Автоматическая раскладка оптимизирует только печать, но следующая операция (ламинация) может не вмещать получившийся лист в пакет плёнки. Пример:

Визитки 90×50 мм + вылеты по 3 мм = 96×56 мм
Автораскладка на печатном листе: помещается 24 шт
        ↓ но
такой плотно заполненный лист НЕ помещается в пакет плёнки для ламинации
        ↓ поэтому
менеджер вручную уменьшает раскладку до 20 шт/лист

Это иллюстрирует общий принцип вложенности форматов: раскладку для печати нужно согласовать с ограничениями постпечати (размер пакета плёнки, поле захвата гильотины). Ручная правка количества позволяет избежать ситуации, когда напечатанный лист физически нельзя обработать на следующей операции.

5. Скорость печати (Konica C6085 / C3070L)

Время печати — отдельная технологическая стадия перед резкой и ламинацией. Скорость зависит от принтера, плотности и типа бумаги, а также (через коэффициент) от формата.

Матрица скоростей печати (printer_speed_matrix: C6085 — 85 стр/мин; C3070L — 71/50/36) и коэффициенты форматов (format_speed_coefficients) — единственный владелец этих данных — описаны в ../06-production/equipment.md, раздел 2 (2.1 «Матрица скоростей печати», 2.2 «Коэффициенты форматов»). Здесь они не дублируются; этот раздел отвечает только за алгоритм расчёта времени печати (PrintTimeCalculator), который читает скорость из матрицы.

6. Ламинация

6.1. Типы ламинации

Тип Логика расчёта Принцип расхода материала
Пакетная материал (пакет) + время Готовые форматы (А3, А4, А5, А6); расход по числу пакетов
Рулонная материал по площади + время Плёнка за кв.м, ширина рулона, учёт отходов

Плёнка бывает разной плотности (микроны), текстуры и финиша: глянец, мат, софт-тач, холст, лён и др. Плёнка хранится в справочнике материалов отдельно от бумаги (категории lamination_packet, lamination_roll).

6.2. Пакетная ламинация

В пакетной ламинации есть готовые форматы (как у бумаги): А3, А4, А5, А6. Резка не нужна, если формат стандартный. Резка требуется при нестандартном размере или когда запаянные края надо обрезать под формат бумаги. Иногда запаянные поля оставляют намеренно — чтобы влага не проникала внутрь и ламинация держалась прочнее.

function calculatePacketLamination(material, format, quantity, needsCutting = false) {
    const materialCost = material.packet_cost * quantity;   // стоимость пакетов
    let timePerUnit = 1; // минута на пакет (загрузка + ламинация)
    let cuttingTime = 0, cuttingCost = 0;
    if (needsCutting) {
        cuttingTime = quantity * 0.5;   // полминуты на пакет
        cuttingCost = cuttingTime * 30; // 30 руб/мин
    }
    const laborCost = (quantity * timePerUnit + cuttingTime) * 30; // 30 руб/мин
    const totalCost = materialCost + laborCost + cuttingCost;
    return { materialCost, laborCost, cuttingCost, totalCost };
}

Реальные форматы плёнки

Плёнка для ламинации больше номинального формата: пакет А3 фактически 303 × 426 мм (для других стандартных размеров плёнка также чуть больше номинала). Эти размеры предлагается хранить в справочнике для точного расчёта последовательных операций.

⚠️ ОТКРЫТЫЙ ВОПРОС: фактический полезный размер пакета А3 указан то как 297×420 (номинал А3), то как 303×426 (реальный размер плёнки). Для корректной раскладки «изделие → пакет» нужно зафиксировать, какой размер использовать при расчёте вложенности.

6.3. Ламинация листовок А5 — вложенность форматов (SRA3 → пакет А3 → рез А5)

Технологическая цепочка: печать на SRA3 → ламинация целого листа в пакет А3 → рез на готовые А5.

Печатный лист SRA3 (320×450) с 4 макетами А5
        ↓
Вставляется в пакет А3 (297×420)
        ↓
Ламинируется целиком
        ↓
Разрезается на 4 готовые листовки А5

Ключевой принцип: количество пакетов = количество печатных листов (ламинируются целые листы, а не отдельные изделия).

Раскладка А5 на SRA3 (А5 148×210 + вылеты 3 мм → 154×216 мм):

  • книжная 154×216: по 320 → 2, по 450 → 2 = 4 шт;
  • альбомная 216×154: по 320 → 1, по 450 → 2 = 2 шт.
  • Выбираем 4 шт. Печатных листов на 100 шт: ceil(100/4) = 25.

6.4. Рулонная ламинация

Расход считается по площади изделия с коэффициентом отходов.

function calculateRollLamination(material, productWidth, productHeight, quantity) {
    const areaPerItem = (productWidth * productHeight) / 1_000_000; // мм² → м²
    const wasteCoefficient = 1.1;                 // +10% на отходы
    const filmPerItem = areaPerItem * wasteCoefficient;
    const totalFilmArea = filmPerItem * quantity;
    const materialCost = totalFilmArea * material.cost_per_sqm;     // плёнка за кв.м
    const laminatorSpeed = 5;                     // м²/мин — скорость ламинатора
    const totalTime = totalFilmArea / laminatorSpeed;
    const laborCost = totalTime * 30;             // 30 руб/мин
    return { materialCost, laborCost, filmArea: totalFilmArea };
}

Учитываются: ширина рулона (мм), плотность (микроны), текстура/финиш, длина рулона (для остатков), стоимость за кв.м, коэффициент отходов 1.1 (+10%), скорость ламинатора ~5 м²/мин.

⚠️ ОТКРЫТЫЙ ВОПРОС: единица скорости ламинатора указана неоднозначно — в комментарии «примерно 5 м/мин», а в формуле используется как «м²/мин» (totalTime = totalFilmArea / 5). Нужно уточнить, измеряется ли скорость в погонных метрах или квадратных метрах в минуту.

6.5. Отображение клиенту

Клиенту не показываются технологические детали (на каком формате идёт ламинация). Клиент выбирает только: оставлять запаянные поля по контуру или обрезать их под формат — с пояснением, как это выглядит и зачем нужно. Формат плёнки подбирается автоматически исходя из этого выбора и раскладки.

Опция обрезки выводится не всегда: для услуги «листовки» по умолчанию плёнка обрезается в край формата (опция скрыта), для услуги «ламинация документов» опция обрезки краёв включается.

7. Универсальная система дополнительных услуг

7.1. Четыре логики расчёта

Тип Логика (calculation_type) Описание
1. Без материалов time_only Биговка, фальцовка, сверление, скругление углов — расход только времени
2. Пакетная ламинация material_packet Готовые форматы + работа; может требовать резки
3. Рулонная ламинация material_roll Расход по площади (кв.м), учёт отходов
4. Сборочные операции assembly Брошюровка (скоба, пружина, клей) — состоят из подопераций

Особенности операций без материалов:

  • скругление углов (разный радиус), сверление (разный диаметр) — особенности на стоимость не влияют;
  • биговка: одно или два сложения — чем больше сложений, тем больше времени; формат изделия на стоимость не влияет.

7.2. Концепция (три уровня → цена)

УРОВЕНЬ 1: Справочник операций — тип услуги, способ расчёта времени, ставка оператора (руб/мин)
        ↓
УРОВЕНЬ 2: Материалы для операций — каждый вариант материала отдельной записью с ценой и расходом
        ↓
УРОВЕНЬ 3: Правила маржи — зависимость от тиража, тип маржи (процент или фиксированная)
        ↓
РЕЗУЛЬТАТ: Цена для клиента = (материал + время) × (1 + маржа%)

7.5. Универсальная формула

Цена для клиента = (Стоимость_материала + Время × Ставка_оператора) × (1 + Маржа%/100)

где:
  Стоимость_материала = расход × цена_материала × количество
  Время               = базовое_время + время_на_единицу × количество
  Маржа%              — из таблицы правил (зависит от тиража)

Ставка оператора по умолчанию: 30 руб/мин. Маржа по умолчанию (если правило не найдено): 50%.

7.6. Пример расчёта (ламинация А3, 50 листов)

1. Материал: 10 руб × 50 = 500 руб
2. Время: 2 + (0.5 × 50) = 27 мин
3. Труд: 27 мин × 30 руб = 810 руб
4. Себестоимость: 500 + 810 = 1310 руб
5. Маржа (тираж 31–50): 70%
6. Цена для клиента: 1310 × 1.7 = 2227 руб
7. Цена за лист: 44.54 руб

7.7. Прайс операций без материалов (формулы)

function calculateTimeOnlyOperation(operation, quantity, complexity = 1) {
    let totalTime = operation.time_per_unit * quantity;
    totalTime *= complexity;                       // коэффициент сложности
    const cost = totalTime * operation.cost_per_minute;
    return { time: totalTime, cost };
}

7.8. Дополнительные параметры заказа (планируемые)

  • Расход на приладку — менеджер может включать или не включать в стоимость заказа.
  • Коэффициент за срочность — добавляется по усмотрению менеджера.
  • Скидка — на конкретную позицию или на весь заказ целиком.

7.9. Согласование цены сайт ↔ менеджер

Клиент на сайте видит цену и соглашается с ней. Если фактическая маржа отличается незначительно — стоимость для клиента не меняют (можно слегка повысить или понизить маржу). Если разница значительна (нестандартный заказ) — стоимость согласовывается менеджером после просчёта всех нюансов. Цель — минимизировать вмешательство менеджера: он только утверждает один из автоматически рассчитанных вариантов и отправляет клиенту.

⚠️ ОТКРЫТЫЙ ВОПРОС: пороги «незначительной» и «значительной» разницы маржи между ценой сайта и расчётом менеджера не заданы количественно — требуется определить правило автоутверждения (например, % отклонения, при котором цена для клиента не меняется).
