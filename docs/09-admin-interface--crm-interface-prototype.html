Прототип интерфейса CRM (менеджерский)

Текстовый прототип менеджерского интерфейса CRM типографии PrintOffice: глобальная структура (header + боковое меню + рабочая область), 8 модулей навигации, детализация рабочих экранов по компонентам, схема взаимодействия модулей, ключевые сценарии работы менеджера и технические требования к UI.

Глобальная структура интерфейса

Интерфейс построен по классической трёхзонной схеме: верхняя панель, боковое меню навигации и рабочая область с динамическим содержимым.

┌─────────────────────────────────────────────────────────────────┐
│  Верхняя панель (Header)                                          │
│  - Логотип / название системы                                     │
│  - Поиск заказа / клиента (глобальный)                            │
│  - Переключение ролей (менеджер / технолог / бухгалтер)           │
│  - Уведомления (колокольчик)                                      │
│  - Профиль менеджера                                              │
├───────────────┬───────────────────────────────────────────────────┤
│               │                                                   │
│  Боковое меню │    Рабочая область (Content)                      │
│  (навигация)  │    - динамическое содержимое                      │
│               │    - вкладки / списки / карточки                  │
│               │                                                   │
└───────────────┴───────────────────────────────────────────────────┘

Верхняя панель (Header)

  • Логотип / название системы.
  • Глобальный поиск заказа или клиента.
  • Переключатель ролей: менеджер / технолог / бухгалтер.
  • Уведомления (колокольчик).
  • Профиль менеджера.

Рабочая область

Динамическое содержимое в зависимости от выбранного модуля: вкладки, списки, карточки.

Боковое меню (навигация)

Восемь основных модулей CRM:

Иконка Модуль Назначение
📊 Дашборд Главная страница с ключевыми показателями
📦 Заказы Список всех заказов, фильтры, статусы
👥 Клиенты База клиентов (B2C / B2B), история заказов
💬 Коммуникации Чаты, email, Telegram, уведомления
🏭 Производство Загрузка оборудования, очередь, ТЗ
📄 Документы Счета, УПД, закрывающие документы
📊 Отчёты Продажи, эффективность менеджеров
⚙️ Настройки Справочники, маржа, конструкторы, партнёры

Компоненты рабочей области (по модулям)

Дашборд

Компонент Описание
KPI-карточки Количество новых заказов, в работе, готовых, просроченных
График заказов По дням / неделям (сумма и количество)
Задачи менеджера Список pending-задач (позвонить, отправить КП, проверить макет)
Последние заказы Таблица с быстрым переходом
Активность клиентов Недавние сообщения, просмотры КП

Список заказов

Компонент Описание
Панель фильтров По статусу, дате, менеджеру, типу печати, клиенту
Поисковая строка По номеру заказа, названию клиента, телефону
Таблица / карточки заказов Колонки: №, клиент, дата, статус, сумма, ответственный
Быстрые действия Кнопки: «Открыть», «Отправить КП», «Счёт», «Чат»
Пагинация Постраничная навигация

Карточка заказа (центральный экран)

Центральный экран работы менеджера. Организован по вкладкам (tabs).

Вкладка Содержимое
Основное Номер заказа, дата, клиент, статус, ответственный менеджер, каналы связи
Состав заказа Список позиций (тип продукции, параметры, количество, цена). Редактирование позиций, удаление, добавление
Финансы Себестоимость, маржа (процент и сумма), итоговая цена. Ручная корректировка цены, скидки, приладка, срочность
Доставка Способ (самовывоз / курьер), адрес, дата и время, стоимость, контакт получателя. Возможность заказать курьера (Достависта / Яндекс)
Коммерческое предложение Кнопка «Сформировать КП»; просмотр отправленных КП, статус (отправлено, просмотрено, принято, отклонено); ссылка для клиента
Чат / Коммуникации История переписки (email, Telegram, WhatsApp, внутренние заметки). Поле ввода, выбор канала, шаблоны ответов
Файлы Список загруженных макетов, скан-копий, договоров. Загрузка нового файла, скачивание, удаление. Превью макета с метками (обрезы, вылеты)
Производство Этапы (печать, резка, ламинация, сборка) со статусами. Комментарии для цеха (печатнику, резчику, оператору плоттера). Параметры резки / раскладки
Документы Счёт на оплату (PDF), УПД, акт. Кнопка «Отправить клиенту»
История изменений Лента: кто и когда изменил статус, цену, добавил комментарий

Блок быстрых действий (вверху карточки)

  • «Обновить статус» (выпадающий список).
  • «Отправить КП».
  • «Выставить счёт».
  • «Создать задачу».
  • «Печать заказа».

Клиенты

Компонент Описание

Фильтры По типу, по постоянству, по городу

Действия Создать заказ для клиента, отправить ссылку на бриф, массовая рассылка

Коммуникации и задачи

Компонент Описание
Единый чат-центр Объединённые сообщения из всех каналов, группировка по клиенту / заказу
Шаблоны ответов Быстрые фразы, переменные (имя клиента, номер заказа, сумма)
Задачи менеджеров Список задач (тип, дедлайн, приоритет). Создание, назначение, закрытие с комментарием
Уведомления Push-уведомления в браузере / Telegram о новых заказах, сообщениях, изменении статуса

Производственный модуль (для менеджера — просмотр)

Для менеджера модуль доступен в режиме просмотра.

Компонент Описание
Календарь загрузки оборудования По дням, по станкам (принтеры, резаки, плоттеры)
Очередь заданий Список активных производственных задач с привязкой к заказам
ТЗ для цеха Детальные инструкции (формат, бумага, раскладка, параметры резки, срок) — можно отправить в Telegram / распечатать
Комментарии операторов Обратная связь от производства (фактическое время, брак, проблемы)

Отчёты

Компонент Описание
Продажи по периодам Графики и таблицы (день, неделя, месяц, квартал)
Топ клиентов По сумме заказов, по количеству
Эффективность менеджеров Количество обработанных заказов, сумма продаж, конверсия
Маржинальность Средняя маржа по типам продукции / технологиям
Каналы привлечения Заказы из сайта, email, Telegram, звонки

Настройки (админ-часть, доступна руководителю)

Компонент Описание
Справочники Оборудование, материалы, форматы, цветность, партнёры
Правила маржи Таблицы наценок (по тиражу, по сумме заказа)
Конструкторы услуг Настройка страниц калькуляторов (какие поля показывать, вылеты, доступные бумаги)
Шаблоны документов Счета, УПД, коммерческие предложения
Пользователи Добавление менеджеров, технологов, бухгалтеров, разграничение прав

Роли пользователей

В системе предусмотрены роли менеджер / технолог / бухгалтер с переключением через header. Гибкая система прав обеспечивает разные меню для разных ролей. Модуль «Настройки» (справочники, маржа, конструкторы, шаблоны документов, пользователи) относится к админ-части и доступен руководителю.

Взаимодействие компонентов (схема)

[Дашборд] ──→ [Список заказов] ──→ [Карточка заказа] ──┬──→ [Чат / Коммуникации]
                                                       ├──→ [Документы / Счёт]
                                                       ├──→ [Производство / ТЗ]
                                                       └──→ [История изменений]

[Клиенты] ──→ [Карточка клиента] ──→ [Создать заказ] ──→ [Карточка заказа]

[Производство] ──→ [Очередь задач] ──→ [ТЗ для цеха] ──→ обратная связь → [Карточка заказа]

[Отчёты] ──→ данные из [Заказы] + [Клиенты] + [Производство]

[Настройки] ──→ влияет на [Справочники], [Маржу], [Конструкторы], [Шаблоны]

Ключевые сценарии работы менеджера

Обработка нового заказа

Уведомление → открыть заказ → проверить параметры → рассчитать цену → сформировать КП → отправить клиенту → создать задачу «перезвонить через 2 часа».

Согласование и оплата

Клиент принимает КП → менеджер выставляет счёт → отправляет ссылку на оплату → после оплаты меняет статус на «Оплачено» → передаёт в производство.

Работа с макетами

Клиент загружает файл → автоматическая проверка (префлайт) → менеджер видит отчёт → при ошибках отправляет клиенту на доработку → после успеха утверждает макет.

Контроль производства

Менеджер видит этапы в карточке заказа → может добавить срочный комментарий резчику → следит за датой готовности.

Закрытие заказа

Готово → менеджер уведомляет клиента → выдача / доставка → формирует УПД → закрывает заказ.

Технические требования к компонентам (для разработки)

  • SPA-подход (React / Vue) с динамической подгрузкой данных через API.
  • WebSocket для живых уведомлений и чата.
  • Адаптивный дизайн (десктоп — приоритет, но планшеты тоже поддерживаются).
  • Гибкая система прав (разные меню для разных ролей).
  • Интеграция с Telegram, Email, курьерскими API.
  • Модульность — каждый компонент можно расширять независимо.
