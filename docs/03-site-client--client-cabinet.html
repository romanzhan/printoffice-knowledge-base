Личный кабинет клиента

Личный кабинет (ЛК) — раздел сайта для постоянных клиентов типографии PrintOffice, где отображаются заказы и коммерческие предложения, доступны история заказов и брифов, повтор заказа в один клик, отслеживание статусов и интерактивное согласование макета с превью.

Назначение и аудитория

ЛК предназначен прежде всего для постоянных клиентов. В нём собраны:

  • Статусы заказов в реальном времени;
  • История заказов с возможностью повторить заказ в один клик;
  • История брифов (заполненных, в процессе, отправленных);
  • Коммерческие предложения / расчёты с интерактивным согласованием;
  • Превью макета с производственными метками, подтверждением корректности или сообщением о проблеме.

Состав личного кабинета

Статусы заказа

Клиент видит текущий статус каждого заказа. Базовый набор статусов, отображаемых клиенту:

Статус Значение
Оплачен Заказ оплачен
В работе Заказ в производстве
Готов Заказ изготовлен
Доставлен Заказ доставлен клиенту

Дополнительно используется статус согласования расчёта (см. ниже): «Ожидает согласования», который после подтверждения клиентом переходит в «Подтверждён», а при отказе клиента — в «Отклонён».

История заказов

В ЛК хранится история всех заказов клиента. По каждому заказу доступны детали и действия.

Повтор заказа в 1 клик

Из истории заказов клиент может повторить ранее оформленный заказ одной кнопкой — параметры предыдущего заказа подставляются автоматически.

Связанные привилегии постоянного клиента

Отметка «постоянный» у клиента открывает дополнительные возможности (вне самого ЛК, но связанные с ним):

  • Возможность отсрочки платежа;
  • Автоматическое создание УПД;
  • Интеграция с Эльбой (API).

Карточка заказа и согласование расчёта (КП)

Для постоянных клиентов в ЛК отображаются КП/заказы. Когда клиент получает расчёт (например, по ссылке вида https://crm.print.ru/p/123), он попадает на интерактивную страницу заказа.

Обратная связь от клиента по расчёту

Перейдя по ссылке, клиент видит интерактивную страницу и может выполнить одно из действий:

  • Согласиться с предложенным вариантом:
    • Автоматически создаётся счёт;
    • Менеджер получает уведомление;
    • Заказ переходит в статус «Подтверждён».
  • Выбрать другой вариант (если предложено несколько):
    • Фиксируется выбор клиента;
    • Пересчитывается итог.
  • Запросить консультацию (вариант «Нужна консультация»).

Клиент может оставить произвольный комментарий к выбору.

Превью макета с метками и подтверждение

В ЛК клиента доступен интерактивный предпросмотр макета с нанесёнными производственными метками (линии реза, сгибы, безопасная зона, вылеты, фольгирование, лак, контурная резка). Клиент проверяет корректность макета и либо подтверждает его, либо сообщает о проблеме.

Интерфейс предпросмотра

┌───────────────────────────────────────────────────────────────────┐
│  ПРЕДПРОСМОТР МАКЕТА                                  [Заказ №12345]│
├───────────────────────────────────────────────────────────────────┤
│  🖼️ ПРЕВЬЮ МАКЕТА                                                  │
│      [ИНТЕРАКТИВНОЕ ПРЕВЬЮ — изображение макета с метками]         │
│      [ + Увеличить ] [ - Уменьшить ] [ Скачать PDF ] [ Скачать PNG]│
│                                                                     │
│  📋 ЛЕГЕНДА                                                        │
│  🟥 Красная линия — линия реза (по ней будет обрезано)            │
│  🟧 Оранжевая пунктирная — линия сгиба                             │
│  🟩 Зеленый пунктир — безопасная зона (важный текст не выходит за) │
│  🔵 Синяя линия — зона вылетов (фон должен доходить до этой линии) │
│  🟡 Золотая область — фольгирование                                │
│  🔷 Голубая область — выборочный лак                               │
│  🟣 Фиолетовая линия — контурная резка                             │
│                                                                     │
│  💡 РЕКОМЕНДАЦИИ                                                   │
│  • Важные элементы (текст, логотип) — внутри зеленой безопасной зоны│
│  • Фоновое изображение должно доходить до синей зоны вылетов        │
│  • Области фольгирования и лака отмечены цветными зонами            │
│                                                                     │
│  [ ПОДТВЕРДИТЬ, ЧТО ВСЕ ВЕРНО ]  [ СООБЩИТЬ О ПРОБЛЕМЕ ]           │
└───────────────────────────────────────────────────────────────────┘

Элементы интерфейса предпросмотра

  • Превью макета — интерактивное изображение с наложенными метками.
  • Управление масштабом — кнопки «Увеличить», «Уменьшить», сброс к 100%. Масштаб ограничен диапазоном 0.5–3 (50%–300%).
  • Скачивание — кнопки «Скачать PDF» и «Скачать PNG».
  • Легенда — расшифровка цветовых обозначений меток.
  • Рекомендации — текстовые подсказки по корректной подготовке макета.
  • Кнопки решения — «Подтвердить, что всё верно» и «Сообщить о проблеме».

Цветовые обозначения и параметры генерации

Легенда цветовых меток (рез, сгиб, безопасная зона, вылеты, фольга, лак, контур), типы меток, класс генератора MarkupPreviewGenerator с параметрами по умолчанию (bleed 3 мм, safeMargin 5 мм, maxSize 600 px), хранение пользовательских меток (таблица order_markups), админ-настройка меток для продуктов и интеграция с калькулятором (MarkupIntegration) описаны в каноническом разделе препресса: Препресс: интерактивное превью с метками.

История брифов в ЛК

В личном кабинете отображается история брифов клиента с их статусами и доступными действиями.

┌───────────────────────────────────────────────────┐
│  ЛИЧНЫЙ КАБИНЕТ | ИСТОРИЯ БРИФОВ                    │
├───────────────────────────────────────────────────┤
│  📋 ЗАКАЗ №12345 | ООО "Ромашка"                   │
│  Бриф на листовку                                  │
│  Статус: ✅ Заполнен                                │
│  Заполнен: 16.03.2026 11:30                         │
│  [ ПРОСМОТРЕТЬ ОТВЕТЫ ] [ СКАЧАТЬ PDF ]            │
├───────────────────────────────────────────────────┤
│  📋 ЗАКАЗ №12344 | ИП Иванов                       │
│  Бриф на логотип                                   │
│  Статус: ⏳ В процессе (60% заполнено)              │
│  Ссылка: [ ПРОДОЛЖИТЬ ЗАПОЛНЕНИЕ ]                 │
└───────────────────────────────────────────────────┘

Статусы и действия по брифам

Статус брифа Доступные действия
✅ Заполнен «Просмотреть ответы», «Скачать PDF»; отображается дата и время заполнения
⏳ В процессе (с процентом заполнения) «Продолжить заполнение» по сохранённой ссылке

Каждая запись в истории привязана к заказу (номер заказа и название клиента/компании) и содержит название брифа (например, «Бриф на листовку», «Бриф на логотип»).

Ссылка на бриф действительна 7 дней с момента отправки. Создание, отправка и обработка брифов (класс BriefManager, таблица brief_instances, отправка по email/Telegram) подробно описаны в документации настройки CRM/сайта (crm-site-setup.md).

⚠️ ОТКРЫТЫЙ ВОПРОС: в карточке заказа в ЛК (CRM) фигурирует домен crm.print.ru, в страницах брифов и ссылках — printoffice.ru. Единый домен личного кабинета и схема его поддоменов (CRM-ЛК и публичный сайт) не зафиксированы.
