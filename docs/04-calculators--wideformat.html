Калькулятор широкоформатной печати

Калькулятор широкоформатной (wide) печати рассчитывает стоимость по формуле «Ширина × Высота × цена за м² + дополнительные услуги». Часть продукции (баннер, плёнка, пластик, люверсы, выборка) поставляется подрядчиком по фиксированному прайсу с наценкой 36 %; для нестандартных конфигураций используется ручной просчёт через Telegram.

Базовая формула расчёта

Основная формула стоимости одного изделия:

стоимость = (Ширина × Высота × Цена_м²) + Допы

Эта формула применяется всегда, без исключений (нюансов нет).

Развёрнутая логика расчёта (псевдокод):

// 1. Расчёт площади
площадь_изделия_м2 = (ширина_мм × высота_мм) / 1_000_000
общая_площадь = площадь_изделия_м2 × количество

// 2. Расчёт стоимости печати
стоимость_печати = общая_площадь × цена_за_м2(материал, разрешение)

// 3. Расчёт люверсов
if (люверсы_по_периметру) {
    периметр_мм = 2 × (ширина_мм + высота_мм)
    количество_люверсов = Math.ceil(периметр_мм / шаг_люверсов_мм) × 2
}
стоимость_люверсов = количество_люверсов × цена_люверса

// 4. Расчёт проклейки краёв
if (проклейка_по_периметру) {
    метраж_проклейки_м = (2 × (ширина_мм + высота_мм)) / 1000
    стоимость_проклейки = метраж_проклейки_м × цена_проклейки_за_м
}

// 5. Расчёт накатки
стоимость_накатки = общая_площадь × цена_накатки_за_м2

// 6. Итоговая цена
итого = стоимость_печати + стоимость_люверсов + стоимость_проклейки +
        стоимость_отверстий + стоимость_карманов + стоимость_накатки +
        стоимость_ламинации + стоимость_резки

Прайс подрядчика (ручной просчёт)

Часть позиций поставляется подрядчиком с фиксированным прайсом. Запрос подрядчику отправляется в виде ТЗ в Telegram, среднее время ответа — около 15 минут. Наценка на wide — обычно 36 %.

⚠️ ОТКРЫТЫЙ ВОПРОС: цена люверсов в прайсе подрядчика указана 15 руб/шт, но в данных калькулятора (таблица доп. услуг и пример расчёта на бэкенде) фигурирует 30 руб/шт. Требуется уточнить актуальную цену.

⚠️ ОТКРЫТЫЙ ВОПРОС: для контурной резки не определено окончательно, считается она за м² или за изделие — нужна единая единица измерения.

Логика расчёта на бэкенде (Product Configurator)

Калькулятор реализуется как интерактивный Product Configurator, а не статичная форма. При изменении любого параметра (тип материала, размеры, тираж, опции) цена пересчитывается без перезагрузки: фронтенд (React/Next.js) дёргает backend API.

Поток расчёта

Фронтенд отправляет запрос с параметрами (например: Баннер, Ш = 1000 мм, В = 2000 мм, Люверсы = 10 шт). Бэкенд выполняет:

  1. Определяет услугу — ищет в таблице vendor_services услугу «Баннер» для wide-подрядчика.
  2. Считает базовую себестоимость:
    • Площадь: 1 м × 2 м = 2 м²
    • Печать: 2 м² × 500 ₽/м² = 1000 ₽
    • Люверсы: 10 шт × 15 ₽/шт = 150 ₽
    • Итого себестоимость: 1150 ₽
  3. Применяет Rule Engine (наценку):
    • Глобальное правило для wide: markup = 36%
    • Цена для клиента = 1150 ₽ × 1.36 = 1564 ₽
  4. Сохраняет в сессию результат расчёта с параметрами и себестоимостью (пока без создания заказа), чтобы не потерять данные при оформлении.

Ответ фронтенду:

{
  "total_price": 1564,
  "breakdown": { "print": 1000, "grommets": 150, "tax": 414 }
}

Оформление заказа

При нажатии «Добавить в корзину» / «Заказать»:

  • Захват данных: минимум — Имя и Телефон (для физлиц) или Email (для юрлиц, чтобы выслать счёт). Можно добавить поле «Загрузить макет».
  • Создание заказа в CRM: статус «Новый (с сайта)»; в заказ подтягиваются рассчитанные параметры и фиксируется себестоимость (1150 ₽).
  • Менеджер видит готовый заказ: «Клиент Иванов хочет баннер 2 м² с люверсами. Себестоимость 1150 ₽, цена для клиента 1564 ₽. Маржа 414 ₽ (36 %)».
  • Связка с макетом: если макет загружен, он прогоняется через префлайт (проверку) и прикрепляется к заказу.

API и БД (технические эндпоинты)

  • POST /api/calculate — принимает параметры, возвращает цену.
  • POST /api/orders — создаёт заказ с привязкой к результатам расчёта.
  • Таблица price_rules — формулы и прайсы.
  • Таблица orders с полем cost_price (себестоимость), чтобы всегда знать маржу.

Риски и нюансы

  • Сложные конфигурации: если нужно несколько баннеров разного размера — давать возможность добавлять несколько позиций в корзину, как в интернет-магазине.
  • Точность себестоимости: цены подрядчика зашиты в систему; нужен механизм быстрого обновления этих данных в админке без программиста.
  • Юридические лица: им может быть важнее получить счёт, а не оплатить картой сразу — система должна предлагать соответствующий флоу.

Структура базы данных

-- 1. Основные продукты (широкоформатная печать)
CREATE TABLE wide_products (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL,
    code VARCHAR(100) UNIQUE NOT NULL,
    description TEXT,
    category VARCHAR(50) NOT NULL,          -- 'printing', 'cutting', 'construction'
    icon_url VARCHAR(500),
    preview_image VARCHAR(500),
    material_id INT,                        -- привязка к материалу (если есть)
    has_contour_cutting BOOLEAN DEFAULT false,
    has_cutting_selection BOOLEAN DEFAULT false,  -- резка с выборкой (буквы)
    price_type ENUM('per_sqm', 'per_item', 'per_meter') DEFAULT 'per_sqm',
    base_price DECIMAL(10,2),
    is_active BOOLEAN DEFAULT true,
    sort_order INT DEFAULT 0,
    FOREIGN KEY (material_id) REFERENCES wide_materials(id)
);

-- 2. Готовые форматы для продуктов (A4, A3, A2, A1, A0)
CREATE TABLE wide_product_formats (
    id INT PRIMARY KEY AUTO_INCREMENT,
    product_id INT NOT NULL,
    format_id INT NOT NULL,                 -- ссылка на wide_ready_formats
    quantity_from INT NOT NULL,
    quantity_to INT,
    price DECIMAL(10,2) NOT NULL,
    is_active BOOLEAN DEFAULT true,
    FOREIGN KEY (product_id) REFERENCES wide_products(id),
    FOREIGN KEY (format_id) REFERENCES wide_ready_formats(id)
);

-- 3. Дополнительные услуги (только действительно доп.)
CREATE TABLE wide_addons (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL,
    code VARCHAR(100) UNIQUE NOT NULL,
    description TEXT,
    hint_text TEXT,                         -- текст подсказки (при нажатии на "?")
    calculation_type ENUM('per_item', 'per_meter', 'per_sqm', 'fixed') NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    is_active BOOLEAN DEFAULT true,
    sort_order INT DEFAULT 0
);

-- 4. Привязка доп. услуг к продуктам
CREATE TABLE wide_product_addons (
    id INT PRIMARY KEY AUTO_INCREMENT,
    product_id INT NOT NULL,
    addon_id INT NOT NULL,
    is_available BOOLEAN DEFAULT true,
    sort_order INT DEFAULT 0,
    FOREIGN KEY (product_id) REFERENCES wide_products(id),
    FOREIGN KEY (addon_id) REFERENCES wide_addons(id)
);

Интерфейс калькулятора

Шаги конфигуратора

  1. Шаг 1. Тип изделия и материал. Выбор типа продукции (Баннер, ПВХ, Плёнка, Бумага, Холст, Сетка), затем конкретного материала, зависящего от типа:
    • Баннер: Китай 440 г/м² / Европейский 510 г/м² / Баннерная ткань / Блэк-Бек / Транслюцентный
    • Плёнка: ORAJET / Китайская / Транслюцентная / Blackout / Перфорированная / Oracal
    • ПВХ: 3 мм / 5 мм / ПВХ с резкой
    • Бумага: Фотобумага 220 гр / Транслюцентная 150 гр
    • Холст: натуральный
    • Сетка: 320 г/м²
  2. Шаг 2. Размеры и тираж. Готовый формат (A4–A0) или свой размер (Ширина × Высота, мм); количество, шт. Автоматически считаются площадь одного изделия (м²) и общая площадь печати (м²).
  3. Шаг 3. Качество печати (разрешение). Выбор dpi (влияет на цену и детализацию): 360 dpi (стандарт, для баннеров на расстоянии), 540 dpi (повышенное качество), 720 dpi (для постеров и интерьерной печати), 1440 dpi (премиум, для фотокачества).
  4. Шаг 4. Дополнительная обработка (см. ниже).

Дополнительная обработка (опции)

  • Люверсы (+ руб/шт): настраиваемый шаг установки (по умолчанию 50 см; в калькуляторе показан пример шага 35 см). Стороны: по периметру / сверху / снизу / слева / справа.
  • Проклейка краёв (+ руб/пог.м): для усиления. Стороны: по периметру / сверху / снизу / слева / справа.
  • Отверстия для крепления (+ руб/отверстие): для ПВХ и жёстких материалов. Количество и расположение (По углам / Сверху / Снизу / По чертежу).
  • Карманы для крепления (+ руб/карман): количество и расположение.
  • Накатка на жёсткую основу:
    • ПВХ 3 мм: +1550 руб/м²
    • ПВХ 5 мм: +2050 руб/м²
    • Пенокартон 5 мм: +1400 руб/м²
    • Поликарбонат 6 мм: +1300 руб/м²
    • Композит: +5200 руб/м²
  • Послепечатная обработка: ламинация (+ руб/м²), плоттерная резка (+ руб/м²), резка с выборкой (+ руб/м²), послепечатная обработка по погонному метру (+ руб/пог.м).

Пример заполнения интерфейса (из макета): свой размер Ширина 500 мм × Высота 700 мм, количество 5 шт; люверсы с шагом 35 см (+ 30 руб/шт); отверстия для крепления — количество 4 шт, расположение «По углам» (+ 20 руб/шт).

Градация цен по количеству (пример)

Пример для A3, ПВХ 3 мм с контурной резкой:

Тираж Цена за шт
1 шт 450 руб/шт
2–4 шт 420 руб/шт
5+ шт 400 руб/шт

Пример итога: 5 шт A3, ПВХ 3 мм с контурной резкой → 5 × 400 = 2000 руб. Срок готовности: 1 день (в примере интерфейса — 15.04.2026). Самовывоз: Москва, ул. Орджоникидзе 11 стр 41 (Пн-Сб 10:00–19:00); доступен расчёт доставки.

Блок расчёта стоимости

Калькулятор выводит детализацию по строкам (площадь × цена/м² или количество × цена) для каждого компонента: печать, люверсы, проклейка краёв, отверстия, карманы, накатка, ламинация, плоттерная резка — и итог + цену за м².

Попап-подсказка (при нажатии на «?»)

Пример подсказки «Проклейка краёв»:

Проклейка краёв — это усиление краёв баннера специальной лентой.

Зачем нужно:

  • Защищает баннер от разрыва в местах установки люверсов
  • Увеличивает срок службы изделия
  • Рекомендуется для больших баннеров (более 3 м) и при ветровых нагрузках

Как рассчитывается:

  • Стоимость = периметр изделия × 50 руб/пог.м
  • Например, для баннера 2×1 м: периметр = 6 м, стоимость = 300 руб
