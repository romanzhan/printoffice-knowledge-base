Клиенты

Модуль управления клиентами CRM PrintOffice: карточка клиента (контакты, мессенджеры, адреса, ИНН, скидка, история заказов), типология клиентов и работа с постоянными клиентами.

Карточка клиента#

Карточка клиента содержит следующие поля.

Поле Описание
ФИО / Название организации Имя физлица или наименование юрлица
Телефон Основной и дополнительный
Email Электронная почта
Никнеймы в мессенджерах Telegram, WhatsApp, Макс
Адрес Фактический и адрес доставки
ИНН Для юридических лиц
История заказов Список заказов с номерами, суммами и статусами
Договор Ссылка на файл
Персональная скидка Опционально

Мессенджеры#

Никнеймы в мессенджерах (Telegram, WhatsApp, Макс) кликабельны, доступно копирование. Перед ответом можно выбрать, с какого аккаунта или номера отвечать клиенту.

Адреса#

В карточке хранятся два типа адресов:

  • Фактический адрес.
  • Адрес доставки.

Реквизиты юрлиц#

Для юридических лиц фиксируется ИНН. ИНН используется в том числе как один из критериев глобального поиска.

Поиск по клиентам#

Глобальная строка поиска расположена вверху экрана и выполняет одновременный поиск по нескольким критериям.

Поиск ведётся по:

  • Номеру заказа (включая буквенные части).
  • Телефону клиента.
  • Email.
  • Никнейму в Telegram / Макс / WhatsApp.
  • Адресу доставки.
  • ИНН.
  • Названию организации.
  • Номеру счёта (из Эльбы).

Результат поиска — список заказов либо переход в карточку клиента.

Постоянные клиенты#

Постоянные клиенты выделяются отдельной отметкой и получают расширенные возможности обслуживания.

  • Отметка «постоянный» → возможность отсрочки платежа.
  • Автоматическое создание УПД (через интеграцию с Эльбой по API).
  • Доступ к истории заказов клиента.
  • Повторный заказ в 1 клик.

Статусы заказа отображаются в личном кабинете клиента: оплачен, в работе, готов, доставлен.

Типы клиентов#

Тип клиента определяется на сайте по наличию cookie, авторизации и параметров URL. Типология используется для адаптации интерфейса и логики оформления заказа.

Определение типа клиента#

Тип Условие
Новый Нет cookie, нет авторизации
Возвращающийся (cookie) Есть cookie returning_user, нет авторизации
Авторизованный Есть JWT-токен в cookies
По ссылке Параметр ref=order&id=12345 в URL

Различия в интерфейсе#

Интерфейс адаптируется в зависимости от того, новый клиент или постоянный.

Элемент Новый клиент Постоянный клиент
Приветствие «Привет! Макет готов?» «С возвращением, Иван!»
Кнопки в герое 3 кнопки (макет / дизайн / не уверен) 3 кнопки (повторить / статус / новый)
Подсказки в калькуляторе Полные Сокращённые или отсутствуют
Предложение офсета Показывать Не показывать (если уже знает)
Финальный CTA Стандартный Сокращённый

B2C / B2B и сегментация заказов#

Признак клиента (новый / постоянный, B2C / B2B) влияет на маршрутизацию заказа между автоматической обработкой (простой заказ) и менеджером (сложный заказ).

Параметр Простой заказ (авто) Сложный заказ (менеджер)
Тип продукции Листовки, визитки, наклейки Брошюры, каталоги, стикерпаки
Тираж До 500 шт От 500 шт
Макет Есть Нет макета
Срочность Обычная Срочная (менее 24 часов)
Клиент Новый Постоянный, B2B
Доп. услуги 1–2 Сложные (вырубка, фальцовка)

Тип клиента (b2b) добавляет балл к оценке сложности заказа в логике принятия решения. Это вторая, отдельная система типизации заказа — балльная оценка сложности (complexityScore) для маршрутизации «авто vs менеджер», не путать с основным автоопределением типа заказа в ядре CRM (класс OrderTypeDetector, признаки isFastB2C / isComplexB2B) — см. CRM: обзор → Типы заказов и их автоопределение. Две системы пока не сведены между собой (см. открытый вопрос ниже).

function getOrderType(orderParams) {
  let complexityScore = 0;
  if (['brochure', 'catalog'].includes(orderParams.productType)) complexityScore += 2;
  if (orderParams.quantity > 500) complexityScore += 1;
  if (!orderParams.hasMaket) complexityScore += 2;
  if (orderParams.isUrgent) complexityScore += 1;
  if (orderParams.clientType === 'b2b') complexityScore += 1;
  return complexityScore >= 3 ? 'complex' : 'simple';
}

⚠️ ОТКРЫТЫЙ ВОПРОС: В источниках используются две разные системы типизации клиента, не сведённые между собой. На сайте тип определяется техническими признаками (новый / возвращающийся по cookie / авторизованный / по ссылке), а в логике сегментации заказов и в карточке CRM применяются бизнес-категории (новый / постоянный, B2C / B2B). Не определено соответствие между технической типизацией и бизнес-категориями, а также способ установки признаков «постоянный» и «B2B» в карточке клиента (вручную менеджером или автоматически по истории заказов).
